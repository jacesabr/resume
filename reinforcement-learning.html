Reinforcement Learning:
DQN & PPO

Teaching agents to make sequential decisions through trial and error. From the Bellman equation to modern policy optimization, covering value-based and policy-gradient methods with full PyTorch implementations.

Mathematical Foundation

Reinforcement learning rests on a handful of foundational equations. Every modern algorithm — from tabular Q-learning to PPO — is a practical approximation of these core ideas.

Bellman Equation for Optimal Value

The optimal value of a state equals the best immediate reward plus the discounted value of the next state. This recursive relationship is the backbone of all value-based RL.

\[ V^*(s) = \max_a \left[ R(s, a) + \gamma \sum_{s'} P(s' | s, a) \, V^*(s') \right] \]

Bellman Equation for Q-Values

The action-value function \( Q(s,a) \) captures the expected return of taking action \( a \) in state \( s \) and then following the optimal policy. DQN directly approximates this function with a neural network.

\[ Q^*(s, a) = R(s, a) + \gamma \sum_{s'} P(s' | s, a) \, \max_{a'} Q^*(s', a') \]

Q-Learning Update Rule

The temporal-difference (TD) update bootstraps from the current estimate. The learning rate \( \alpha \) controls step size, and the term in brackets is the TD error — the surprise signal that drives learning.

\[ Q(s, a) \leftarrow Q(s, a) + \alpha \Big[ r + \gamma \max_{a'} Q(s', a') - Q(s, a) \Big] \]

Policy Gradient Theorem

Instead of estimating values, policy gradient methods directly optimize the policy parameters \( \theta \). The gradient of the expected return points in the direction that makes high-reward actions more probable.

\[ \nabla_\theta J(\theta) = \mathbb{E}_{\pi_\theta} \Big[ \nabla_\theta \log \pi_\theta(a|s) \, A^{\pi}(s, a) \Big] \]

PPO Clipped Surrogate Objective

PPO constrains how much the policy can change in a single update. The ratio \( r_t(\theta) \) measures the probability shift, and the clip function prevents destructively large updates — achieving the stability of trust region methods without the computational cost.

\[ L^{CLIP}(\theta) = \mathbb{E}_t \Big[ \min \big( r_t(\theta) \, \hat{A}_t, \; \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) \, \hat{A}_t \big) \Big] \]

where \( r_t(\theta) = \frac{\pi_\theta(a_t | s_t)}{\pi_{\theta_\text{old}}(a_t | s_t)} \) and \( \epsilon \) is typically 0.2.

Generalized Advantage Estimation (GAE)

GAE provides a smooth trade-off between bias and variance in advantage estimates. The parameter \( \lambda \) interpolates between one-step TD (low variance, high bias) and Monte Carlo (high variance, low bias).

\[ \hat{A}_t^{GAE(\gamma,\lambda)} = \sum_{l=0}^{\infty} (\gamma \lambda)^l \, \delta_{t+l} \quad \text{where} \quad \delta_t = r_t + \gamma V(s_{t+1}) - V(s_t) \]

The RL Framework

Reinforcement learning formalizes the problem of an agent learning to act in an environment through interaction. Unlike supervised learning, there is no labeled dataset — the agent discovers what works through trial and error, guided only by scalar reward signals.

The Agent-Environment Loop

At each discrete time step \( t \), the agent observes the current state \( s_t \), selects an action \( a_t \) according to its policy \( \pi \), receives a reward \( r_t \) from the environment, and transitions to a new state \( s_{t+1} \). The objective is to learn a policy that maximizes the expected cumulative discounted reward (the return).

Agent

The decision-maker. It maintains a policy \( \pi(a|s) \) — a mapping from states to actions (or a probability distribution over actions). The agent's goal is to improve this policy over time.

Environment

Everything outside the agent. It receives actions, produces next states and rewards. The agent does not need to know the environment's internal dynamics — it learns from experience alone.

State & Action Spaces

State space \( \mathcal{S} \): the set of all possible observations (pixel arrays, sensor readings, game boards). Action space \( \mathcal{A} \): discrete (left/right/fire) or continuous (torque, velocity).

Reward & Return

The reward \( r_t \) is a scalar feedback signal. The return \( G_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k} \) is the discounted sum of future rewards. The discount factor \( \gamma \in [0,1) \) encodes how much the agent cares about the future vs. the present.

Policy

A policy \( \pi \) can be deterministic \( a = \mu(s) \) or stochastic \( a \sim \pi(\cdot|s) \). Value-based methods (DQN) derive the policy implicitly from Q-values. Policy-gradient methods (PPO) parameterize and optimize it directly.

Value Function

The state-value function \( V^\pi(s) \) estimates expected return from state \( s \) under policy \( \pi \). The action-value function \( Q^\pi(s,a) \) estimates expected return from taking action \( a \) in state \( s \). The advantage \( A(s,a) = Q(s,a) - V(s) \) measures how much better an action is relative to the average.

The Loop in Practice

1. Environment resets → agent receives initial state \( s_0 \).
2. Agent picks action \( a_0 \sim \pi(\cdot|s_0) \).
3. Environment returns reward \( r_0 \) and next state \( s_1 \).
4. Agent stores transition \( (s_0, a_0, r_0, s_1) \).
5. Agent updates its policy/value using stored experience.
6. Repeat until the episode terminates (done=True) or a time limit is reached.

Architecture Overview

DQN Architecture — Value-Based

DQN approximates the optimal action-value function \( Q^*(s,a) \) with a neural network. The network takes the state as input and outputs a Q-value for each possible action. The agent acts greedily by selecting the action with the highest Q-value (with epsilon-greedy exploration).

Input Layer

State representation. For CartPole: 4-dimensional vector (cart position, velocity, pole angle, angular velocity). For Atari: 84x84 grayscale frames stacked 4-deep, processed through convolutional layers.

Hidden Layers

Feature extraction. Typically 2–3 fully connected layers with ReLU activations (128–512 units each). For pixel inputs, 3 convolutional layers (32, 64, 64 filters) followed by a fully connected layer.

Output Layer

Q-values. One output neuron per discrete action. No activation function — Q-values are unbounded real numbers. The agent picks \( \arg\max_a Q(s,a) \).

Target Network

Stability mechanism. A frozen copy of the Q-network, updated every \( C \) steps. The TD target uses the target network's Q-values, preventing the "moving target" problem that destabilizes training.

Actor-Critic Architecture — PPO

PPO uses two function approximators that typically share a feature backbone. The actor (policy network) outputs a probability distribution over actions. The critic (value network) estimates the state-value function \( V(s) \) used to compute advantages.

Shared Backbone

Both actor and critic process the state through shared layers. This parameter sharing improves sample efficiency — the feature extractor is trained by both policy and value gradients simultaneously.

Actor Head

For discrete actions: outputs logits passed through softmax to produce action probabilities. For continuous actions: outputs mean \( \mu \) and log-std \( \log\sigma \) of a Gaussian distribution. Actions are sampled as \( a = \mu + \sigma \cdot \epsilon, \; \epsilon \sim \mathcal{N}(0,1) \).

Critic Head

A single linear output neuron producing the scalar state-value estimate \( \hat{V}(s) \). Trained with MSE loss against the return-to-go or GAE targets. Never directly used for action selection.

Training Loop

PPO collects a batch of trajectories, computes advantages via GAE, then runs multiple epochs of minibatch SGD on the clipped objective. The ratio between new and old action probabilities is clipped to \( [1-\epsilon, 1+\epsilon] \) to prevent policy collapse.

Key Concepts

Markov Decision Process (MDP)

The mathematical formalism behind RL. An MDP is defined by the tuple \( (\mathcal{S}, \mathcal{A}, P, R, \gamma) \). The Markov property states that the future depends only on the current state, not the history: \( P(s_{t+1} | s_t, a_t) \). All RL algorithms assume the environment is (at least approximately) an MDP.

Exploration vs. Exploitation

The fundamental dilemma. Exploitation: act greedily according to current knowledge. Exploration: try new actions to discover better strategies. Too little exploration and the agent gets stuck in local optima. Too much and it never converges. Epsilon-greedy, Boltzmann exploration, and intrinsic curiosity are common solutions.

Epsilon-Greedy Strategy

With probability \( \epsilon \), select a random action; otherwise, select the greedy action \( \arg\max_a Q(s,a) \). Epsilon typically decays from 1.0 to 0.01 over training. Simple but effective — ensures every action has a nonzero chance of being selected, guaranteeing asymptotic convergence in the tabular case.

Experience Replay

Store transitions \( (s, a, r, s', \text{done}) \) in a replay buffer (typically 100K–1M entries). During training, sample random minibatches from the buffer. This breaks temporal correlations between consecutive samples, dramatically improving stability and sample efficiency. Introduced by Mnih et al. (2015) in the original DQN paper.

Target Network

A separate copy of the Q-network whose parameters are updated infrequently (hard update every \( C \) steps) or slowly (soft update: \( \theta^- \leftarrow \tau\theta + (1-\tau)\theta^- \)). Without it, the TD target \( r + \gamma \max Q(s',a'; \theta) \) shifts every gradient step, causing divergent oscillation. The target network provides a stable regression target.

Advantage Function

\( A(s,a) = Q(s,a) - V(s) \). It measures how much better an action is compared to the average action in that state. Using advantages instead of raw returns reduces variance in policy gradient estimates without introducing bias. The critic network estimates \( V(s) \), and the advantage is computed from TD errors.

Clipped Surrogate Objective

PPO's core innovation. The probability ratio \( r_t(\theta) = \pi_\theta(a|s) / \pi_{\theta_\text{old}}(a|s) \) is clipped to \( [1-\epsilon, 1+\epsilon] \). If the advantage is positive, the objective caps the ratio at \( 1+\epsilon \) (preventing over-commitment). If negative, it caps at \( 1-\epsilon \). This creates a pessimistic lower bound on policy improvement.

Generalized Advantage Estimation

GAE computes advantages as an exponentially-weighted average of multi-step TD errors. \( \lambda = 0 \) gives one-step TD (low variance, high bias). \( \lambda = 1 \) gives Monte Carlo returns (high variance, low bias). In practice, \( \lambda = 0.95 \) provides a good balance. GAE is the standard advantage estimator in PPO, A2C, and most modern policy gradient methods.

Code: DQN for CartPole

A complete Deep Q-Network implementation in PyTorch with experience replay, target network, and epsilon-greedy exploration. This agent learns to balance a pole on a cart, achieving the maximum score of 500 within a few hundred episodes.

Python — dqn_cartpole.py 
import gymnasium as gym
import torch
import torch.nn as nn
import torch.optim as optim
import numpy as np
import random
from collections import deque

# ── Hyperparameters ──────────────────────────────────────────
GAMMA          = 0.99      # discount factor
LR             = 1e-3      # learning rate
BATCH_SIZE     = 64
BUFFER_SIZE    = 100_000
EPSILON_START  = 1.0
EPSILON_END    = 0.01
EPSILON_DECAY  = 0.995
TARGET_UPDATE  = 10        # sync target network every N episodes
NUM_EPISODES   = 500

# ── Q-Network ────────────────────────────────────────────────
class QNetwork(nn.Module):
    def __init__(self, state_dim, action_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim, 128),
            nn.ReLU(),
            nn.Linear(128, 128),
            nn.ReLU(),
            nn.Linear(128, action_dim)
        )

    def forward(self, x):
        return self.net(x)

# ── Replay Buffer ────────────────────────────────────────────
class ReplayBuffer:
    def __init__(self, capacity):
        self.buffer = deque(maxlen=capacity)

    def push(self, state, action, reward, next_state, done):
        self.buffer.append((state, action, reward, next_state, done))

    def sample(self, batch_size):
        batch = random.sample(self.buffer, batch_size)
        states, actions, rewards, next_states, dones = zip(*batch)
        return (
            torch.FloatTensor(np.array(states)),
            torch.LongTensor(actions),
            torch.FloatTensor(rewards),
            torch.FloatTensor(np.array(next_states)),
            torch.FloatTensor(dones)
        )

    def __len__(self):
        return len(self.buffer)

# ── DQN Agent ────────────────────────────────────────────────
class DQNAgent:
    def __init__(self, state_dim, action_dim):
        self.action_dim = action_dim
        self.epsilon = EPSILON_START

        # Online network (learns) + Target network (provides stable targets)
        self.q_net     = QNetwork(state_dim, action_dim)
        self.target_net = QNetwork(state_dim, action_dim)
        self.target_net.load_state_dict(self.q_net.state_dict())

        self.optimizer = optim.Adam(self.q_net.parameters(), lr=LR)
        self.buffer    = ReplayBuffer(BUFFER_SIZE)

    def select_action(self, state):
        # Epsilon-greedy: explore with probability epsilon
        if random.random() < self.epsilon:
            return random.randint(0, self.action_dim - 1)
        with torch.no_grad():
            state_t = torch.FloatTensor(state).unsqueeze(0)
            q_vals  = self.q_net(state_t)
            return q_vals.argmax(dim=1).item()

    def train_step(self):
        if len(self.buffer) < BATCH_SIZE:
            return

        states, actions, rewards, next_states, dones = \
            self.buffer.sample(BATCH_SIZE)

        # Current Q-values for chosen actions
        q_values = self.q_net(states).gather(1, actions.unsqueeze(1)).squeeze()

        # Target Q-values: r + gamma * max_a' Q_target(s', a')
        with torch.no_grad():
            next_q = self.target_net(next_states).max(dim=1)[0]
            targets = rewards + GAMMA * next_q * (1 - dones)

        # Huber loss (smooth L1) — less sensitive to outliers than MSE
        loss = nn.functional.smooth_l1_loss(q_values, targets)

        self.optimizer.zero_grad()
        loss.backward()
        # Gradient clipping for stability
        nn.utils.clip_grad_norm_(self.q_net.parameters(), 1.0)
        self.optimizer.step()

    def sync_target(self):
        self.target_net.load_state_dict(self.q_net.state_dict())

    def decay_epsilon(self):
        self.epsilon = max(EPSILON_END, self.epsilon * EPSILON_DECAY)

# ── Training Loop ────────────────────────────────────────────
env   = gym.make("CartPole-v1")
agent = DQNAgent(
    state_dim  = env.observation_space.shape[0],
    action_dim = env.action_space.n
)

for ep in range(NUM_EPISODES):
    state, _ = env.reset()
    total_reward = 0

    while True:
        action = agent.select_action(state)
        next_state, reward, terminated, truncated, _ = env.step(action)
        done = terminated or truncated

        agent.buffer.push(state, action, reward, next_state, float(done))
        agent.train_step()

        state = next_state
        total_reward += reward

        if done:
            break

    agent.decay_epsilon()

    if ep % TARGET_UPDATE == 0:
        agent.sync_target()

    if ep % 20 == 0:
        print(f"Episode {ep} | Reward: {total_reward:.0f} | Eps: {agent.epsilon:.3f}")

env.close()
print("Training complete.")

Code: PPO with Stable-Baselines3

Stable-Baselines3 provides battle-tested implementations of major RL algorithms. This example shows how to train PPO on both discrete (CartPole) and continuous (LunarLander) environments with proper evaluation, logging, and model persistence.

Python — ppo_sb3.py 
import gymnasium as gym
from stable_baselines3 import PPO
from stable_baselines3.common.env_util import make_vec_env
from stable_baselines3.common.evaluation import evaluate_policy
from stable_baselines3.common.callbacks import EvalCallback, CheckpointCallback

# ── 1. Vectorized environments for parallel rollout collection ──
# SB3 runs N envs in parallel to fill the rollout buffer faster
train_env = make_vec_env("CartPole-v1", n_envs=4)
eval_env  = gym.make("CartPole-v1")

# ── 2. Configure PPO ─────────────────────────────────────────
model = PPO(
    policy          = "MlpPolicy",       # 2-layer MLP (64, 64)
    env             = train_env,
    learning_rate   = 3e-4,             # Adam LR
    n_steps         = 2048,             # rollout length per env
    batch_size      = 64,               # minibatch size for SGD
    n_epochs        = 10,               # epochs per rollout update
    gamma           = 0.99,             # discount factor
    gae_lambda      = 0.95,             # GAE lambda
    clip_range      = 0.2,              # PPO epsilon clip
    ent_coef        = 0.01,             # entropy bonus (exploration)
    vf_coef         = 0.5,              # value function loss weight
    max_grad_norm   = 0.5,              # gradient clipping
    verbose         = 1,
    tensorboard_log = "./ppo_logs/"
)

# ── 3. Callbacks for evaluation & checkpointing ──────────────
eval_cb = EvalCallback(
    eval_env,
    best_model_save_path = "./best_model/",
    log_path             = "./eval_logs/",
    eval_freq            = 5000,
    n_eval_episodes      = 10,
    deterministic        = True
)

ckpt_cb = CheckpointCallback(
    save_freq  = 10_000,
    save_path  = "./checkpoints/",
    name_prefix = "ppo_cartpole"
)

# ── 4. Train ─────────────────────────────────────────────────
model.learn(
    total_timesteps = 100_000,
    callback        = [eval_cb, ckpt_cb],
    progress_bar    = True
)

# ── 5. Evaluate final model ──────────────────────────────────
mean_reward, std_reward = evaluate_policy(model, eval_env, n_eval_episodes=20)
print(f"Final evaluation: {mean_reward:.1f} +/- {std_reward:.1f}")

# ── 6. Save & reload ─────────────────────────────────────────
model.save("ppo_cartpole_final")

# Later: load and deploy
loaded_model = PPO.load("ppo_cartpole_final")

# ── 7. Watch the trained agent ────────────────────────────────
env = gym.make("CartPole-v1", render_mode="human")
obs, _ = env.reset()

for _ in range(1000):
    action, _ = loaded_model.predict(obs, deterministic=True)
    obs, reward, terminated, truncated, info = env.step(action)
    if terminated or truncated:
        obs, _ = env.reset()

env.close()

# ═══════════════════════════════════════════════════════════════
# Continuous control example: LunarLander
# ═══════════════════════════════════════════════════════════════
cont_env = make_vec_env("LunarLanderContinuous-v3", n_envs=8)

cont_model = PPO(
    "MlpPolicy",
    cont_env,
    learning_rate = 3e-4,
    n_steps       = 1024,
    batch_size    = 64,
    n_epochs      = 4,
    clip_range    = 0.2,
    gae_lambda    = 0.98,        # higher lambda for continuous
    ent_coef      = 0.0,         # no entropy bonus needed
    verbose       = 1
)
cont_model.learn(total_timesteps=500_000, progress_bar=True)
cont_model.save("ppo_lunar_continuous")

Code: Custom Actor-Critic (A2C) from Scratch

A from-scratch Advantage Actor-Critic implementation in PyTorch. This builds the shared-backbone architecture, collects rollouts, computes GAE advantages, and updates both the actor (policy) and critic (value) heads. Understanding this code gives you the foundation for implementing PPO — the only additions are the probability ratio and the clip.

Python — a2c_scratch.py 
import gymnasium as gym
import torch
import torch.nn as nn
import torch.optim as optim
from torch.distributions import Categorical
import numpy as np

# ── Hyperparameters ──────────────────────────────────────────
GAMMA       = 0.99
GAE_LAMBDA  = 0.95
LR          = 2.5e-4
ENT_COEF    = 0.01     # entropy bonus encourages exploration
VF_COEF     = 0.5      # value loss weight
ROLLOUT_LEN = 128      # steps before each update
NUM_UPDATES = 1000

# ── Actor-Critic Network ─────────────────────────────────────
class ActorCritic(nn.Module):
    """Shared backbone with separate actor and critic heads."""

    def __init__(self, state_dim, action_dim):
        super().__init__()

        # Shared feature extractor
        self.backbone = nn.Sequential(
            nn.Linear(state_dim, 256),
            nn.Tanh(),
            nn.Linear(256, 128),
            nn.Tanh()
        )

        # Policy head → action probabilities
        self.actor = nn.Linear(128, action_dim)

        # Value head → scalar state value V(s)
        self.critic = nn.Linear(128, 1)

        # Orthogonal initialization (improves RL training)
        for layer in self.modules():
            if isinstance(layer, nn.Linear):
                nn.init.orthogonal_(layer.weight, gain=np.sqrt(2))
                nn.init.zeros_(layer.bias)

    def forward(self, x):
        features = self.backbone(x)
        logits   = self.actor(features)
        value    = self.critic(features).squeeze(-1)
        return logits, value

    def get_action_and_value(self, state):
        """Sample action, return log_prob, entropy, and value."""
        logits, value = self.forward(state)
        dist   = Categorical(logits=logits)
        action = dist.sample()
        return action, dist.log_prob(action), dist.entropy(), value

    def evaluate(self, states, actions):
        """Re-evaluate stored actions under current policy."""
        logits, values = self.forward(states)
        dist = Categorical(logits=logits)
        return dist.log_prob(actions), dist.entropy(), values

# ── GAE Computation ──────────────────────────────────────────
def compute_gae(rewards, values, dones, next_value):
    """Generalized Advantage Estimation.

    Computes advantages as exponentially-weighted sum of TD errors:
        A_t = sum_{l=0}^{T-t} (gamma * lambda)^l * delta_{t+l}
    where delta_t = r_t + gamma * V(s_{t+1}) - V(s_t)
    """
    advantages = torch.zeros_like(rewards)
    gae = 0.0

    for t in reversed(range(len(rewards))):
        if t == len(rewards) - 1:
            next_val = next_value
        else:
            next_val = values[t + 1]

        # TD error: r + gamma * V(s') - V(s)
        delta = rewards[t] + GAMMA * next_val * (1 - dones[t]) - values[t]

        # Accumulate GAE with decay
        gae = delta + GAMMA * GAE_LAMBDA * (1 - dones[t]) * gae
        advantages[t] = gae

    returns = advantages + values
    return advantages, returns

# ── Training Loop ────────────────────────────────────────────
env   = gym.make("CartPole-v1")
model = ActorCritic(
    state_dim  = env.observation_space.shape[0],
    action_dim = env.action_space.n
)
optimizer = optim.Adam(model.parameters(), lr=LR, eps=1e-5)

state, _ = env.reset()
ep_reward   = 0
ep_rewards  = []

for update in range(NUM_UPDATES):
    # ── Collect rollout ──────────────────────────────────────
    states, actions, rewards, dones = [], [], [], []
    log_probs, values, entropies   = [], [], []

    for step in range(ROLLOUT_LEN):
        state_t = torch.FloatTensor(state).unsqueeze(0)

        with torch.no_grad():
            action, lp, ent, val = model.get_action_and_value(state_t)

        next_state, reward, terminated, truncated, _ = env.step(action.item())
        done = terminated or truncated

        states.append(state)
        actions.append(action.item())
        rewards.append(reward)
        dones.append(float(done))
        log_probs.append(lp)
        values.append(val)
        entropies.append(ent)

        ep_reward += reward
        state = next_state

        if done:
            ep_rewards.append(ep_reward)
            ep_reward = 0
            state, _ = env.reset()

    # ── Compute advantages via GAE ────────────────────────────
    states_t  = torch.FloatTensor(np.array(states))
    actions_t = torch.LongTensor(actions)
    rewards_t = torch.FloatTensor(rewards)
    dones_t   = torch.FloatTensor(dones)
    values_t  = torch.cat(values)

    # Bootstrap from last state
    with torch.no_grad():
        _, next_value = model(torch.FloatTensor(state).unsqueeze(0))

    advantages, returns = compute_gae(
        rewards_t, values_t.detach(), dones_t, next_value.item()
    )

    # Normalize advantages (reduces variance)
    advantages = (advantages - advantages.mean()) / (advantages.std() + 1e-8)

    # ── Compute losses ────────────────────────────────────────
    new_lp, new_ent, new_vals = model.evaluate(states_t, actions_t)

    # Policy loss: -log_prob * advantage (maximize expected advantage)
    policy_loss = -(new_lp * advantages.detach()).mean()

    # Value loss: MSE between predicted values and GAE returns
    value_loss = nn.functional.mse_loss(new_vals, returns.detach())

    # Entropy bonus: encourages exploration by penalizing certainty
    entropy_loss = -new_ent.mean()

    # Combined loss
    loss = policy_loss + VF_COEF * value_loss + ENT_COEF * entropy_loss

    optimizer.zero_grad()
    loss.backward()
    nn.utils.clip_grad_norm_(model.parameters(), 0.5)
    optimizer.step()

    # ── Logging ──────────────────────────────────────────────
    if update % 25 == 0 and ep_rewards:
        avg = np.mean(ep_rewards[-10:])
        print(f"Update {update:4d} | Avg Reward: {avg:6.1f} | Policy Loss: {policy_loss:.4f}")

env.close()
print(f"Training complete. Best avg: {max(
    [np.mean(ep_rewards[i:i+10]) for i in range(len(ep_rewards)-9)]
    ) if len(ep_rewards) >= 10 else 0:.1f}")

Algorithm Landscape

The RL algorithm space is broadly divided into value-based methods (learn Q-values, derive policy implicitly), policy-gradient methods (optimize the policy directly), and actor-critic methods (combine both). Here is where the major algorithms fall.

Value-Based · Tabular

Q-Learning

The original off-policy TD algorithm. Maintains a Q-table \( Q(s,a) \) for every state-action pair. Converges to optimal Q-values given sufficient exploration. Limitation: only works for small, discrete state spaces. Cannot generalize to unseen states.

Value-Based · Deep

DQN

Deep Q-Network (Mnih et al., 2015). Replaces the Q-table with a neural network. Key innovations: experience replay and target network. First algorithm to achieve human-level performance on Atari games from raw pixels. Works only with discrete actions.

Value-Based · Deep

Double DQN

Addresses DQN's overestimation bias. Standard DQN uses the same network to select and evaluate actions, inflating Q-values. Double DQN decouples selection (online net picks the best action) from evaluation (target net evaluates it). Simple change, significant improvement.

Actor-Critic · On-Policy

A2C

Advantage Actor-Critic. Synchronous version of A3C. The actor learns the policy, the critic learns the value function. Uses the advantage \( A = Q - V \) to reduce gradient variance. On-policy — data is discarded after each update, making it less sample-efficient but more stable.

Actor-Critic · On-Policy

PPO

Proximal Policy Optimization (Schulman et al., 2017). The de facto standard for policy optimization. Clips the probability ratio to prevent large policy updates. Achieves trust-region-like stability with first-order optimization. Used to train ChatGPT (RLHF), robotic locomotion, and game AI.

Actor-Critic · Off-Policy

SAC

Soft Actor-Critic (Haarnoja et al., 2018). Maximizes a maximum entropy objective: reward + entropy bonus. The entropy term encourages exploration and produces robust, multi-modal policies. Off-policy with a replay buffer. State-of-the-art for continuous control (robotics, locomotion).

Actor-Critic · Off-Policy

TD3

Twin Delayed DDPG (Fujimoto et al., 2018). Builds on DDPG with three fixes: twin Q-networks (take the minimum to reduce overestimation), delayed policy updates (update actor less frequently than critic), and target policy smoothing (add noise to target actions). Strong baseline for continuous control.

Choosing an Algorithm

Decision Heuristic

Discrete actions? Start with DQN or Double DQN. Continuous actions? Start with PPO (simplicity) or SAC (sample efficiency). Need RLHF? PPO is the standard. Sim-to-real robotics? SAC or TD3. Massively parallel envs? PPO with vectorized rollouts.

When to Use / When to Avoid

When to Use RL

Sequential decision-making — the problem involves a series of actions over time, not a single prediction.

Delayed rewards — the payoff of an action is not immediately clear (chess moves, treatment plans, resource allocation).

No labeled data — you can define a reward function but cannot label optimal actions for every state.

Simulation available — you have a simulator (game, physics engine, market model) where the agent can safely explore millions of trajectories.

Adaptive control — the system dynamics are unknown, nonlinear, or change over time (robotics, autonomous driving, HVAC optimization).

RLHF for LLMs — aligning language models to human preferences (PPO on reward models trained from human comparisons).

When to Avoid RL

Labeled data exists — if you have (input, correct_output) pairs, supervised learning will be faster, cheaper, and more reliable.

No simulator — RL requires millions of interactions. If each interaction is expensive or dangerous in the real world (surgery, nuclear reactors), pure RL is impractical without sim-to-real transfer.

Reward shaping is hard — poorly designed reward functions cause reward hacking (the agent optimizes the metric without achieving the goal). If you cannot define a clear reward, RL will fail.

Stateless problems — if the decision does not depend on a sequence of prior states (image classification, regression), RL adds unnecessary complexity.

Real-time safety constraints — RL agents must explore, which means they will take bad actions during training. If safety violations are unacceptable, use constrained RL or a different paradigm.

Small, deterministic problems — if the environment is small enough to enumerate, dynamic programming or classical search (A*, BFS) will solve it exactly and far more efficiently.

Best Practices

  • Start with a baseline. Random policy, then a simple heuristic, then RL. If the heuristic is already good, RL may not be worth the engineering cost.
  • Normalize observations and rewards. Neural networks train better when inputs are zero-mean, unit-variance. Clip rewards to [-10, 10] or use reward normalization.
  • Use vectorized environments. Collect data from 8-64 parallel environments. This is the single biggest throughput improvement for on-policy methods.
  • Monitor entropy. If the policy's entropy collapses to zero too quickly, the agent has stopped exploring. Increase the entropy coefficient or use curiosity-driven exploration.
  • Seed everything. RL training is notoriously noisy. Run at least 3-5 seeds and report mean/std of returns, not a single cherry-picked run.
  • Log everything. Use TensorBoard or Weights & Biases to track episode returns, policy loss, value loss, entropy, KL divergence, and gradient norms. RL debugging is nearly impossible without good logging.

Common Pitfalls

  • Reward hacking. The agent finds a loophole in the reward function. A boat-racing agent learned to drive in circles collecting power-ups instead of finishing the race. Always sanity-check learned behavior.
  • Hyperparameter sensitivity. RL algorithms are more sensitive to hyperparameters than supervised learning. Learning rate, clip range, GAE lambda, and entropy coefficient all interact nonlinearly. Use established defaults from the papers.
  • Catastrophic forgetting. On-policy methods (PPO, A2C) discard old data. If the environment distribution shifts (new levels, curriculum learning), the agent can forget previous skills.
  • Sparse rewards. If the agent only receives reward at the end of a long episode, credit assignment becomes extremely difficult. Use reward shaping, curriculum learning, or hindsight experience replay.
  • Overestimation bias. DQN systematically overestimates Q-values. Use Double DQN, clipped double-Q (TD3/SAC), or ensemble methods to mitigate.
