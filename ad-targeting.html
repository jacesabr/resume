Ad Targeting & CTR Prediction

Predicting which users will click which ads — the billion-dollar classification problem powering digital advertising.

8 sections · LR baseline, LightGBM, Wide & Deep, position bias, A/B testing

The Problem: Predicting Click-Through Rate (CTR)

Which users will click this ad? Ad targeting is a massive-scale binary classification problem. Typical CTR is 0.1-3%, creating natural class imbalance. The model must predict P(click | user, ad, context) in <10ms at millions of requests/second.

Scale

A mid-sized ad network serves ~1 billion impressions/day. At 2% CTR, that's 20M clicks vs 980M non-clicks. Model must score candidates in <10ms to meet real-time bidding deadlines.

Revenue Impact

1% improvement in CTR prediction → millions in additional annual revenue. The difference between showing a relevant ad vs an irrelevant one is the entire business model.

Mathematical Foundation

\[\text{CTR} = P(\text{click} \mid \text{user}, \text{ad}, \text{context})\]

The core prediction — probability of click given user features, ad features, and context (time, device, page).

\[\text{Revenue} = \text{Bid} \times P(\text{click}) \times P(\text{convert} \mid \text{click})\]

Expected revenue per impression — the ranking function for ad selection. This is the eCPM formula.

\[\text{LogLoss} = -\frac{1}{N}\sum[y\log(\hat{p}) + (1-y)\log(1-\hat{p})]\]

Primary metric — calibrated probabilities matter more than accuracy. LogLoss penalizes confident wrong predictions.

\[\text{AUC-ROC} > 0.75 \text{ is strong for CTR prediction}\]

ROC-AUC measures ranking quality — can the model rank clickers above non-clickers?

"kw">class WideAndDeep(nn.Module):
    "kw">class="kw">def __init__(self, n_wide, n_embed_features, embed_dims, deep_hidden):
        super().__init__()
        # Wide component: linear model on crossed features
        self.wide = nn.Linear(n_wide, 1)
        
        # Deep component: embeddings → MLP
        self.embeddings = nn.ModuleList([
            nn.Embedding(n_cats, dim) 
            "kw">class="kw">for n_cats, dim "kw">class="kw">in zip(n_embed_features, embed_dims)

        # Deep: generalization via learned embeddings

        deep_in = torch.cat(embeds, dim=1)
        deep_out = self.deep(deep_in)
        "kw">class="kw">return torch.sigmoid(wide_out + deep_out)

Failure Mode ✗ — Position Bias & Feedback Loops

Ads in position 1 get more clicks regardless of relevance. If you train on click data without correcting for position, the model learns 'position 1 = good ad' instead of 'relevant ad = good ad'. This creates a feedback loop where top-ranked ads keep getting shown because they were top-ranked.

Key Takeaways & Production Checklist

When This Approach Works

✅ LogLoss over accuracy — calibrated probabilities directly determine bid prices
✅ LightGBM/XGBoost for tabular CTR data — handles categoricals, fast inference
✅ Feature interactions matter most: user×ad category, device×ad format, time×user
✅ A/B test every model change — offline gains don't always translate to revenue
✅ Correct for position bias — raw clicks confound relevance with placement

Common Pitfalls

❌ Training on raw clicks without position bias correction
❌ Optimizing for clicks instead of conversions (clickbait problem)
❌ Ignoring calibration — if predicted 2% CTR is really 5%, bidding math breaks
❌ Cold start: new users/ads have no history — need content-based fallbacks
❌ Feedback loops: model shows ad → gets clicks → reinforces showing same ad
