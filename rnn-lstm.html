RNNs & LSTMs

Recurrent architectures that learn temporal dependencies in sequential data — from text and speech to time series and music.

Vanilla RNN, LSTM, GRU · Gate equations · 3 full code examples · MathJax notation

Mathematical Foundation

Recurrent networks maintain a hidden state that is updated at each timestep, allowing them to model sequential dependencies. The three main variants — Vanilla RNN, LSTM, and GRU — differ in how they compute and propagate this state.

Vanilla RNN
\[h_t = \tanh(W_{hh} \cdot h_{t-1} + W_{xh} \cdot x_t + b_h)\]

Hidden state update: combines previous hidden state \(h_{t-1}\) with current input \(x_t\) through learned weight matrices, squashed by tanh to keep values in \([-1, 1]\).

\[\hat{y}_t = \text{softmax}(W_{hy} \cdot h_t + b_y)\]

Output at each timestep: linear projection of hidden state into output space, normalized via softmax for classification tasks.

LSTM (Long Short-Term Memory)
\[f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)\]

Forget gate: decides what fraction of the old cell state to retain. Sigmoid output in \([0, 1]\): 0 = forget entirely, 1 = keep everything.

\[i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)\]
\[\tilde{C}_t = \tanh(W_C \cdot [h_{t-1}, x_t] + b_C)\]

Input gate: \(i_t\) controls how much of the candidate values \(\tilde{C}_t\) to write into cell state. The candidate is a tanh-squashed proposal of new information.

\[C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t\]

Cell state update: the core innovation. Old state is scaled by the forget gate, then new candidate information is added. The \(\odot\) operator is element-wise multiplication. This is the "highway" that lets gradients flow unimpeded across many timesteps.

\[o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)\]
\[h_t = o_t \odot \tanh(C_t)\]

Output gate: filters which parts of the cell state to expose as the hidden state. Cell state is squashed through tanh then gated by the sigmoid output.

GRU (Gated Recurrent Unit)
\[z_t = \sigma(W_z \cdot [h_{t-1}, x_t] + b_z)\]
\[r_t = \sigma(W_r \cdot [h_{t-1}, x_t] + b_r)\]

Update gate \(z_t\) and Reset gate \(r_t\): the GRU merges forget and input gates into a single update gate, and uses a reset gate to control how much past hidden state bleeds into the candidate.

\[\tilde{h}_t = \tanh(W_h \cdot [r_t \odot h_{t-1}, x_t] + b_h)\]
\[h_t = (1 - z_t) \odot h_{t-1} + z_t \odot \tilde{h}_t\]

Hidden state update: interpolates between old hidden state and candidate. No separate cell state — fewer parameters than LSTM, often similar performance.

The Vanishing Gradient Problem

In a vanilla RNN, gradients are propagated backwards through time (BPTT) by repeatedly multiplying by the recurrent weight matrix \(W_{hh}\). After \(T\) timesteps, the gradient includes a product of \(T\) Jacobian matrices:

\[\frac{\partial L}{\partial h_0} = \frac{\partial L}{\partial h_T} \prod_{t=1}^{T} \frac{\partial h_t}{\partial h_{t-1}} = \frac{\partial L}{\partial h_T} \prod_{t=1}^{T} W_{hh}^T \cdot \text{diag}(\tanh'(h_t))\]

Since \(\tanh'(x) \in (0, 1]\), each multiplication shrinks the gradient. Over 50–100 timesteps, the gradient decays exponentially toward zero. The network cannot learn that a word at position 5 is critical for a prediction at position 95.

Conversely, if the spectral radius of \(W_{hh}\) exceeds 1, gradients explode toward infinity. Gradient clipping mitigates explosion, but does nothing for vanishing.

Why Vanilla RNNs Fail

Repeated multiplication by values < 1 causes exponential decay. After ~20 timesteps, gradients are effectively zero. The network has a "memory horizon" of roughly 10–20 steps — far too short for paragraphs, music, or financial series.

How LSTM Solves It

The cell state acts as a gradient highway. The update rule \(C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t\) is additive, not multiplicative. When the forget gate is near 1.0, gradients pass through unchanged across hundreds of timesteps — no decay.

The Core Intuition

Think of it like a game of telephone. In a vanilla RNN, the message gets distorted at every step (multiplicative). In an LSTM, the message is written on a whiteboard (cell state) and only selectively erased or updated — it can persist indefinitely.

Architecture Overview

RNN Unrolled Through Time
  Input sequence:     x_1         x_2         x_3         x_4        x_T
                      |           |           |           |           |
                      v           v           v           v           v
                 +---------+ +---------+ +---------+ +---------+ +---------+
  Hidden   h_0 ->|  RNN    |->|  RNN    |->|  RNN    |->|  RNN    |->|  RNN    |-> h_T
  state         |  Cell   | |  Cell   | |  Cell   | |  Cell   | |  Cell   |
                 +---------+ +---------+ +---------+ +---------+ +---------+
                      |           |           |           |           |
                      v           v           v           v           v
  Outputs:          y_1         y_2         y_3         y_4        y_T

    * All cells share the SAME weights (W_hh, W_xh, W_hy)
    * h_0 is typically initialized to zeros
    * "Unrolling" = copying the cell T times for backprop
LSTM Cell Internal Structure
                       Cell State (C_{t-1}) ---- Highway ---->
                              |                                    |
                     +--------+--------+                  +--------+--------+
                     |                 |                  |                 |
                     v                 v                  v                 v
              +-------------+   +-------------+   +-------------+   +-------------+
   x_t ----->| Forget Gate |   | Input Gate  |   |  Cell State  |   | Output Gate |
   h_{t-1} ->|  f_t = sig  |   | i_t = sig   |   |   Update    |   |  o_t = sig  |
              +------+------+   +------+------+   +------+------+   +------+------+
                     |                 |                  |                 |
                     v                 v                  v                 v
                f_t * C_{t-1}   +   i_t * C~_t   =      C_t           o_t * tanh(C_t)
                (what to forget)   (what to add)     (new cell)        = h_t (output)

  Key:  sig = sigmoid(0..1)     tanh = tanh(-1..1)     * = element-wise multiply

    Gate parameters:  W_f, W_i, W_C, W_o  (each has own weight matrix + bias)
    Total params per LSTM layer = 4 * ((input_dim + hidden_dim) * hidden_dim + hidden_dim)
GRU Cell (Simplified LSTM)
   x_t -----+----------+-------------------------+
   h_{t-1} -+----+     |                         |
                |    |     |                         |
                v    v     v                         v
         +------------+ +------------+        +------------+
         | Update     | | Reset      |        | Candidate  |
         | z_t = sig  | | r_t = sig  | -----> | h~_t=tanh  |
         +------+-----+ +------------+        +------+-----+
                |                                     |
                v                                     v
         h_t = (1 - z_t) * h_{t-1}  +  z_t * h~_t

    * No separate cell state — only hidden state
    * 2 gates instead of 3 — ~25% fewer parameters than LSTM
    * Often matches LSTM performance on shorter sequences

Key Concepts Glossary

Hidden State (h_t)
The network's "working memory" at timestep t. Encodes a compressed representation of all inputs seen so far. In vanilla RNNs this is the only state; in LSTMs it is derived from the cell state via the output gate.
Cell State (C_t)
LSTM-exclusive. A separate memory vector that runs as a "conveyor belt" through time. Information is added or removed only through gates, enabling long-range memory without gradient decay. This is the key innovation that solved the vanishing gradient problem.
Forget Gate (f_t)
Sigmoid layer that outputs values in [0,1] for each element of the cell state. Decides what information to discard. A value of 0 means "erase completely," 1 means "keep entirely." Initialized with a positive bias (~1.0) so the default behavior is to remember.
Input Gate (i_t)
Controls what new information is written to the cell state. Works in tandem with a tanh candidate layer: the gate scales how much of the candidate to add. This two-part mechanism prevents unbounded growth of cell values.
Output Gate (o_t)
Filters the cell state before exposing it as the hidden state. The cell may store information relevant to future timesteps that should not affect the current output. The output gate handles this selective exposure.
Sequence-to-Sequence (Seq2Seq)
Architecture with an encoder RNN and decoder RNN. The encoder compresses an input sequence into a fixed context vector; the decoder generates an output sequence from it. Used in machine translation (e.g., English to French), text summarization, and chatbots.
Many-to-One
Architecture where an entire input sequence is consumed and only the final hidden state is used for prediction. Classic use case: sentiment analysis (read full review, output positive/negative). The LSTM processes all tokens, then the last h_T feeds a dense classifier.
Bidirectional RNN/LSTM
Runs two separate RNN layers: one processing the sequence left-to-right, one right-to-left. Their hidden states are concatenated at each timestep, giving the model access to both past and future context. Essential for NER, POS tagging, and any task where the whole sequence is available at inference time.

Code Example 1: LSTM for Sentiment Analysis

A many-to-one LSTM trained on the IMDB movie review dataset. The model reads 25,000 reviews (max 500 tokens each), processes them through an embedding layer and LSTM, then classifies sentiment as positive or negative via a single sigmoid output.

Why this architecture works
Sentiment depends on the full context of a review. An LSTM can learn that "not bad" is positive despite containing "bad," because the hidden state carries the negation forward. The final hidden state encodes the aggregate sentiment of the entire sequence.
Python — Keras / TensorFlow
import numpy as np
from tensorflow.keras.datasets import imdb
from tensorflow.keras.preprocessing.sequence import pad_sequences
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Embedding, LSTM, Dense, Dropout
from tensorflow.keras.callbacks import EarlyStopping

# ── Hyperparameters ──────────────────────────────────
VOCAB_SIZE   = 10000   # top 10k most frequent words
MAX_LEN      = 500     # pad/truncate reviews to 500 tokens
EMBED_DIM    = 128     # embedding vector size
LSTM_UNITS   = 64      # LSTM hidden state dimension
DROPOUT_RATE = 0.3     # regularization
BATCH_SIZE   = 64
EPOCHS       = 10

# ── Load & preprocess IMDB ───────────────────────────
(X_train, y_train), (X_test, y_test) = imdb.load_data(
    num_words=VOCAB_SIZE
)

# Pad sequences so all inputs have the same length
X_train = pad_sequences(X_train, maxlen=MAX_LEN, padding='post', truncating='post')
X_test  = pad_sequences(X_test,  maxlen=MAX_LEN, padding='post', truncating='post')

print(f"Training samples: {X_train.shape[0]}")
print(f"Test samples:     {X_test.shape[0]}")
print(f"Sequence length:  {MAX_LEN}")

# ── Build Model ──────────────────────────────────────
model = Sequential([
    # Learns dense vector for each word index
    Embedding(
        input_dim=VOCAB_SIZE,
        output_dim=EMBED_DIM,
        input_length=MAX_LEN
    ),

    # LSTM reads the full sequence, returns only final h_T
    LSTM(
        units=LSTM_UNITS,
        return_sequences=False,   # many-to-one: only last state
        dropout=DROPOUT_RATE,       # dropout on inputs
        recurrent_dropout=0.2      # dropout on recurrent state
    ),

    Dropout(DROPOUT_RATE),

    # Binary classification: positive (1) or negative (0)
    Dense(1, activation='sigmoid')
])

model.compile(
    optimizer='adam',
    loss='binary_crossentropy',
    metrics=['accuracy']
)

model.summary()

# ── Train ────────────────────────────────────────────
early_stop = EarlyStopping(
    monitor='val_loss',
    patience=2,
    restore_best_weights=True
)

history = model.fit(
    X_train, y_train,
    epochs=EPOCHS,
    batch_size=BATCH_SIZE,
    validation_split=0.2,
    callbacks=[early_stop],
    verbose=1
)

# ── Evaluate ─────────────────────────────────────────
loss, accuracy = model.evaluate(X_test, y_test, verbose=0)
print(f"\nTest Accuracy: {accuracy:.4f}")
print(f"Test Loss:     {loss:.4f}")

# ── Predict on a new review ──────────────────────────
word_index = imdb.get_word_index()

def predict_sentiment(text):
    tokens = [word_index.get(w, 2) for w in text.lower().split()]
    tokens = [t if t < VOCAB_SIZE else 2 for t in tokens]
    padded = pad_sequences([tokens], maxlen=MAX_LEN, padding='post')
    score  = model.predict(padded, verbose=0)[0][0]
    label  = "Positive" if score > 0.5 else "Negative"
    return f"{label} ({score:.3f})"

print(predict_sentiment("This movie was absolutely fantastic and deeply moving"))
print(predict_sentiment("Terrible waste of time with awful acting"))

Expected Results

  • Test accuracy: ~86-88% (competitive with CNNs for text classification)
  • Training time: ~3 minutes per epoch on GPU, ~15 min on CPU
  • Total parameters: ~1.36M (embedding: 1.28M, LSTM: 49K, dense: 65)

Code Example 2: Time Series Prediction

An LSTM trained to predict future temperature from historical daily readings. The model uses a sliding window of 60 days to predict the next day's temperature. This same pattern applies to stock prices, energy demand, sensor data, and any univariate or multivariate time series.

Windowing strategy
Time series prediction requires converting a flat sequence into (X, y) pairs using a sliding window. For a window of size 60: X = days [0..59], y = day [60]. The window slides by 1 each step, creating overlapping training samples. LSTM excels here because temperature today depends on patterns from weeks ago (seasonal lag).
Python — Keras / TensorFlow
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.preprocessing import MinMaxScaler
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import LSTM, Dense, Dropout
from tensorflow.keras.callbacks import EarlyStopping

# ── Generate synthetic temperature data ──────────────
# (Replace with pd.read_csv('your_data.csv') for real data)
np.random.seed(42)
days = 1500
t = np.arange(days)
# Seasonal pattern + trend + noise
temp = 20 + 10 * np.sin(2 * np.pi * t / 365) + 0.005 * t + np.random.normal(0, 2, days)

# ── Normalize to [0, 1] ──────────────────────────────
scaler = MinMaxScaler()
temp_scaled = scaler.fit_transform(temp.reshape(-1, 1))

# ── Create sliding windows ───────────────────────────
WINDOW = 60   # look back 60 days

def create_sequences(data, window):
    X, y = [], []
    for i in range(len(data) - window):
        X.append(data[i : i + window])
        y.append(data[i + window])
    return np.array(X), np.array(y)

X, y = create_sequences(temp_scaled, WINDOW)

# ── Train/test split (time series: NO shuffle) ───────
split = int(len(X) * 0.8)
X_train, X_test = X[:split], X[split:]
y_train, y_test = y[:split], y[split:]

print(f"X_train shape: {X_train.shape}")  # (1152, 60, 1)
print(f"X_test  shape: {X_test.shape}")   # (288, 60, 1)

# ── Build stacked LSTM ───────────────────────────────
model = Sequential([
    # First LSTM: return full sequence for stacking
    LSTM(50, return_sequences=True, input_shape=(WINDOW, 1)),
    Dropout(0.2),

    # Second LSTM: return only final state
    LSTM(50, return_sequences=False),
    Dropout(0.2),

    # Regression output: predict next day's temperature
    Dense(25, activation='relu'),
    Dense(1)   # linear activation for regression
])

model.compile(optimizer='adam', loss='mse', metrics=['mae'])

# ── Train ────────────────────────────────────────────
early_stop = EarlyStopping(
    monitor='val_loss', patience=5, restore_best_weights=True
)

history = model.fit(
    X_train, y_train,
    epochs=50,
    batch_size=32,
    validation_split=0.15,
    callbacks=[early_stop],
    verbose=1
)

# ── Predict & inverse transform ──────────────────────
predictions = model.predict(X_test)
predictions = scaler.inverse_transform(predictions)
actual       = scaler.inverse_transform(y_test)

# ── Evaluate ─────────────────────────────────────────
mae = np.mean(np.abs(predictions - actual))
rmse = np.sqrt(np.mean((predictions - actual) ** 2))
print(f"MAE:  {mae:.2f}°C")
print(f"RMSE: {rmse:.2f}°C")

# ── Plot ─────────────────────────────────────────────
plt.figure(figsize=(14, 5))
plt.plot(actual, label='Actual Temperature', alpha=0.8)
plt.plot(predictions, label='LSTM Prediction', alpha=0.8)
plt.xlabel('Day')
plt.ylabel('Temperature (°C)')
plt.title('LSTM Time Series Prediction')
plt.legend()
plt.tight_layout()
plt.show()

Expected Results

  • MAE: ~1.5-2.5°C on synthetic data (real weather data is noisier)
  • Key insight: Stacked LSTMs (2+ layers) capture hierarchical temporal features. Layer 1 learns short-term patterns, Layer 2 learns longer-term trends.
  • Critical rule: Never shuffle time series data. Temporal order must be preserved in train/test split.

Code Example 3: Bidirectional LSTM for NER

Named Entity Recognition (NER) is a sequence-to-sequence labeling task: given a sentence, tag each token as a person, organization, location, or "other." A Bidirectional LSTM is ideal because the correct label for a word often depends on both preceding and following words — e.g., "Washington" could be a person or a location depending on context.

Why bidirectional?
In "Apple released a new iPhone," a forward-only LSTM reading "Apple" doesn't yet know "released" follows. A backward LSTM already has "released" context. Concatenating both directions gives each timestep full sentence context — essential for disambiguation in NER.
Python — Keras / TensorFlow
import numpy as np
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import (
    Embedding, Bidirectional, LSTM, Dense,
    TimeDistributed, Dropout
)
from tensorflow.keras.preprocessing.sequence import pad_sequences
from tensorflow.keras.utils import to_categorical

# ── Example NER data (BIO tagging scheme) ────────────
# In production, load CoNLL-2003 or OntoNotes dataset
sentences = [
    [("John",   "B-PER"), ("lives",  "O"),     ("in",     "O"),
     ("London", "B-LOC"), (".",      "O")],
    [("Apple",  "B-ORG"), ("Inc",    "I-ORG"), ("hired",  "O"),
     ("Sarah",  "B-PER"), ("from",   "O"),     ("Paris",  "B-LOC")],
    [("The",    "O"),     ("UN",     "B-ORG"), ("met",    "O"),
     ("in",     "O"),     ("Geneva", "B-LOC"), ("today",  "O")],
]

# ── Build vocabularies ───────────────────────────────
words = sorted(set(w for s in sentences for w, _ in s))
tags  = sorted(set(t for s in sentences for _, t in s))

word2idx = {w: i + 2 for i, w in enumerate(words)}  # 0=PAD, 1=UNK
tag2idx  = {t: i for i, t in enumerate(tags)}

VOCAB_SIZE = len(word2idx) + 2
NUM_TAGS   = len(tag2idx)
MAX_LEN    = 50

print(f"Vocabulary: {VOCAB_SIZE} words, {NUM_TAGS} tags")
print(f"Tags: {tag2idx}")

# ── Encode sentences ─────────────────────────────────
X = [[word2idx.get(w, 1) for w, _ in s] for s in sentences]
y = [[tag2idx[t]          for _, t in s] for s in sentences]

X = pad_sequences(X, maxlen=MAX_LEN, padding='post', value=0)
y = pad_sequences(y, maxlen=MAX_LEN, padding='post', value=tag2idx['O'])
y = np.array([to_categorical(seq, num_classes=NUM_TAGS) for seq in y])

# ── Build BiLSTM Model ───────────────────────────────
model = Sequential([
    # Word embeddings
    Embedding(
        input_dim=VOCAB_SIZE,
        output_dim=64,
        input_length=MAX_LEN,
        mask_zero=True   # ignore padding in LSTM computation
    ),

    # Bidirectional LSTM: reads forwards AND backwards
    # Output dim = 2 * 64 = 128 (concat of both directions)
    Bidirectional(
        LSTM(64, return_sequences=True, dropout=0.3)
    ),

    Dropout(0.25),

    # Second BiLSTM layer for deeper features
    Bidirectional(
        LSTM(32, return_sequences=True, dropout=0.3)
    ),

    # TimeDistributed: apply Dense to EACH timestep independently
    # Output shape: (batch, MAX_LEN, NUM_TAGS)
    TimeDistributed(
        Dense(NUM_TAGS, activation='softmax')
    )
])

model.compile(
    optimizer='adam',
    loss='categorical_crossentropy',
    metrics=['accuracy']
)

model.summary()

# ── Train ────────────────────────────────────────────
model.fit(
    X, y,
    epochs=50,
    batch_size=2,
    validation_split=0.1,
    verbose=1
)

# ── Predict on new sentence ──────────────────────────
idx2tag = {v: k for k, v in tag2idx.items()}

def predict_entities(sentence_words):
    encoded = [word2idx.get(w, 1) for w in sentence_words]
    padded  = pad_sequences([encoded], maxlen=MAX_LEN, padding='post')
    preds   = model.predict(padded, verbose=0)[0]
    tags    = [idx2tag[np.argmax(p)] for p in preds[:len(sentence_words)]]
    return list(zip(sentence_words, tags))

result = predict_entities(["John", "lives", "in", "London"])
for word, tag in result:
    print(f"  {word:12s} -> {tag}")

# Output:
#   John         -> B-PER
#   lives        -> O
#   in           -> O
#   London       -> B-LOC

Architecture Highlights

  • Bidirectional: concatenates forward and backward hidden states, doubling the feature dimension at each timestep
  • TimeDistributed(Dense): applies the same classification layer to every position, producing a tag for every token
  • mask_zero=True: tells the LSTM to skip padding tokens during computation (crucial for variable-length sequences)
  • Production improvement: add a CRF layer on top (keras-crf) for globally optimal label sequences

Parameter Count Formula

LSTM: 4 * [(input_dim + hidden_dim) * hidden_dim + hidden_dim]
GRU: 3 * [(input_dim + hidden_dim) * hidden_dim + hidden_dim]
Vanilla RNN: (input_dim + hidden_dim) * hidden_dim + hidden_dim
The 4x / 3x multiplier comes from the number of gate weight matrices.

When GRU Beats LSTM

Smaller datasets: GRU has fewer parameters, so it overfits less on small data. Shorter sequences: the GRU's simpler gating is sufficient. Real-time inference: 25% fewer operations per step. In practice, try both and pick whichever validates better on your task.

When to Use / When to Avoid

When to Use RNNs / LSTMs

Sequential data with temporal dependencies: text, speech, music, time series, sensor streams

Variable-length inputs where CNNs would require fixed windows or wasteful padding

Language modeling and text generation where token-by-token decoding is needed

Real-time streaming applications where you process data one step at a time (unidirectional)

Small-to-medium datasets where transformers would overfit due to massive parameter counts

Edge deployment where model size matters: a 2-layer LSTM uses ~500K params vs. millions for a transformer

When to Avoid

Very long sequences (>2000 tokens): even LSTMs struggle; transformers with attention handle these better

Large-scale NLP (2020+): pre-trained transformers (BERT, GPT) dominate in accuracy on most benchmarks

Embarrassingly parallel tasks: RNNs are sequential by nature and cannot parallelize across timesteps during training

Image classification: CNNs and Vision Transformers are vastly superior for spatial data

Tabular data: gradient boosting (XGBoost, LightGBM) almost always outperforms RNNs on structured data

When interpretability is critical: attention weights in transformers are more interpretable than LSTM hidden states

The Practical Reality (2024+)

  • Transformers have largely replaced LSTMs in NLP tasks where compute is available and datasets are large
  • LSTMs remain competitive in time series forecasting, on-device inference, and low-resource language tasks
  • Hybrid architectures (LSTM + attention, or LSTM inside transformer blocks) are actively researched
  • Understanding LSTMs is foundational — the gating mechanism directly inspired transformer attention, and the vanishing gradient problem frames all modern architecture decisions

Common Pitfalls

  • Shuffling time series data: destroys temporal structure. Always split chronologically.
  • Forgetting to scale inputs: LSTMs are sensitive to input magnitude. Always normalize to [0,1] or standardize.
  • Too many stacked layers: 2-3 LSTM layers is usually optimal. Beyond that, gradients degrade and training slows dramatically.
  • Ignoring sequence length: padding to the length of the longest sample wastes computation. Use bucketing or masking.
  • Using bidirectional for autoregressive tasks: you cannot look at future tokens when generating text one word at a time.
