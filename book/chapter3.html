3.1

The Residual Stream

The right mental model for a transformer is not a pipeline — it is a residual stream. A single d-dimensional vector starts as a token embedding, passes through every layer, and exits as the representation used to predict the next token. Each attention head and FFN sublayer reads from this stream and adds its output back. Nothing is overwritten; information accumulates additively.

Elhage et al. (2021, Anthropic) formalised this with a clean mathematical notation. Let x₀ be the embedded tokens, and let each block l read from xₗ₋₁ and write a correction back:

RESIDUAL STREAM — FORMAL NOTATION (ELHAGE ET AL. 2021) x₀ = W_E · tokens
xₗ = xₗ₋₁ + attnₗ(xₗ₋₁) + mlpₗ(xₗ₋₁)
logits = W_U · x_L W_E = embedding matrix | W_U = unembedding matrix | L = number of layers

This view has a deep implication: every component writes to different subspaces of the same shared stream. Elhage et al. show you can decompose each attention head into two independent computations — a QK circuit (which positions attend to which) and an OV circuit (what information is copied when attending). These circuits compose across layers, creating increasingly complex computations through the same stream.

The residual structure is also why transformers train stably at 96+ layers. The gradient of the loss with respect to early layer parameters includes an additive identity term — a direct path that doesn't pass through any matrix multiplication. Without it, gradients decay exponentially with depth and training stalls.

Key insight: Think of the residual stream as a shared whiteboard. The embedding writes the starting position. Each attention head and FFN reads the whiteboard, computes a small update, and writes it back. The unembedding layer reads the final state. Nobody erases the whiteboard — every early signal persists to the end.

3.2

Token Embeddings & the Vocabulary

Before any computation can begin, the model must convert discrete tokens — integers from 0 to 50,256 — into continuous vectors it can do linear algebra on. The embedding matrix W_E is a lookup table of shape [vocab_size, d_model]. Token ID 0 maps to row 0, token ID 42 maps to row 42. For GPT-2 small: [50,257 × 768] = 38.6 million parameters. That's 31% of the model's total 124M, just for this one table.

Proximity in embedding space encodes semantic similarity. After training, king − man + woman ≈ queen holds geometrically. The embedding for "Paris" and "Lyon" are close; "Paris" and "photosynthesis" are far. These relationships emerge purely from predicting next tokens on enough text — no supervised labels needed.

There's a counterintuitive paradox: antonyms like "hot" and "cold" end up close in embedding space. They appear in identical contexts ("the weather was very ___"), so the model learns similar representations for both. Embeddings capture distributional context, not semantic meaning in the human sense.

The vocabulary padding trick: GPT-2's BPE tokenizer produces exactly 50,257 tokens. nanoGPT uses 50,304 instead — 47 phantom tokens that are never used. Why? 50,304 = 786 × 64 — the nearest multiple of 64 above 50,257. The LM head is a matrix multiply of shape [seq_len, 768] × [768, vocab_size]. When vocab_size is a multiple of 64, CUDA tensor cores operate at full warp occupancy on this matmul. Karpathy reported ~25% training speedup from this single change with zero modeling cost.

Key insight: A multiple-of-64 dimension isn't a modeling choice — it's a hardware alignment trick. CUDA GPU warps process groups of 32 threads; tensor core tiles are 8×8 or 16×16 elements. An unaligned dimension forces the kernel to pad internally with partial warps, wasting compute. Pad it yourself to the nearest multiple of 64 and the compiler takes a fully-optimised path.

3.3

Positional Embeddings

Self-attention has no inherent notion of word order. The sentence "the cat sat on the mat" and "the mat sat on the cat" produce identical Q, K, V vectors if processed without position information — the model would predict the same next token from both. To break this symmetry, a positional vector is added to each token embedding before the first block.

The original Vaswani et al. transformer used sinusoidal encodings — fixed functions of position and dimension. For position pos and dimension i:

SINUSOIDAL POSITIONAL ENCODING — VASWANI ET AL. 2017 PE(pos, 2i) = sin(pos / 10000^(2i / d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i / d_model)) Wavelengths range from 2π (dim 0, finest) to 10,000×2π (dim d_model/2, coarsest) — log-spaced over the full range

Why base 10,000? The wavelengths form a geometric progression from 2π ≈ 6.28 positions (dimension 0) to 62,832 positions (dimension d_model/2). This log-spaced distribution means the model can distinguish nearby tokens (via fast-frequency dimensions) and track long-range position (via slow-frequency dimensions) simultaneously from the same encoding.

The sinusoidal choice has a remarkable mathematical property: for any fixed offset k, PE(pos+k) is a linear function of PE(pos). There exists a rotation matrix R_k (block-diagonal, 2×2 rotations in each dimension pair) such that PE(pos+k) = R_k · PE(pos). This means the model can learn to attend to "k positions ahead" by learning a linear transformation of positional encodings — relative position is encoded in a linearly accessible way.

GPT-2 uses learned absolute embeddings instead: a separate W_PE matrix of shape [1024, 768] where each row is a trainable vector for that position. This costs 1024 × 768 = 786,432 parameters (less than 1% of total) but cannot extrapolate to sequences longer than the training context. Modern architectures (LLaMA, Mistral) use RoPE (Rotary Positional Embeddings) — a hybrid that rotates Q and K vectors directly by the position angle, enabling relative position awareness and extrapolation beyond the training length.

Key insight: Sinusoidal PE's base 10,000 is not arbitrary — it's chosen so that, at the maximum training length (512 tokens for the original transformer), the slowest-frequency dimension completes only ~0.05 of a full cycle. This ensures all dimensions still distinguish positions meaningfully, rather than repeating. The same principle scales: LLaMA-3 uses base 500,000 to handle 128K+ context windows.

import math

def sinusoidal_pe(pos, d_model=512, base=10000):
    """Generate the positional encoding vector for a given position."""
    pe = []
    for i in range(d_model):
        freq = math.pow(base, -2 * (i//2) / d_model)
        if i % 2 == 0:
            pe.append(math.sin(pos * freq))
        else:
            pe.append(math.cos(pos * freq))
    return pe

3.4

Layer Normalisation

Activations in deep networks drift. After 12 matrix multiplications, a vector that started with values near 0 and 1 can contain values ranging from −500 to +800. This "internal covariate shift" makes training unstable — the gradient signal from the loss is calibrated to one scale of activations, but receives a completely different scale on the next batch. Layer normalisation (Ba et al. 2016) solves this by normalising each vector to zero mean and unit variance before each sublayer.

LAYER NORMALISATION — BA ET AL. 2016 μ = (1/d) Σᵢ xᵢ    σ² = (1/d) Σᵢ (xᵢ − μ)²    x̂ᵢ = (xᵢ − μ) / √(σ² + ε)
LN(x) = γ ⊙ x̂ + β γ (scale) and β (shift) are learnable per-dimension parameters. ε = 1e-5 prevents division by zero.

RMSNorm (Zhang & Sennrich 2019), used by LLaMA, drops the mean subtraction entirely: it normalises only by the root mean square RMS(x) = √((1/d)Σxᵢ²). The mean subtraction removes a degree of freedom that empirically matters less than the scaling — RMSNorm is about 7–10% faster than LayerNorm at inference while matching quality on downstream tasks.

Pre-norm vs post-norm: The original Vaswani 2017 transformer applied LayerNorm after the residual addition. GPT-2 and all modern LLMs apply it before the sublayer (pre-norm). The difference matters: with post-norm, the residual stream is normalised and the raw-scale bypass is lost. With pre-norm, the sublayer always sees a normalised input while the residual stream carries its full magnitude. Pre-norm trains more stably at depth, especially past 24 layers.

Key insight: The γ and β parameters are the only parts of LayerNorm that are learned — they let the model "undo" the normalisation if needed. Initialised to γ=1, β=0 (identity), they adapt to give each dimension the scale and shift that's most useful for the next sublayer.

▶ LayerNorm vs RMSNorm — implement and compare from scratch
import math

def layer_norm(x, gamma=None, beta=None, eps=1e-5):
    """Standard LayerNorm: subtract mean, divide by std, scale+shift."""
    n = len(x)
    mu  = sum(x) / n
    var = sum((v - mu)**2 for v in x) / n
    xhat = [(v - mu) / math.sqrt(var + eps) for v in x]
    if gamma is None: gamma = [1.0]*n
    if beta  is None: beta  = [0.0]*n
    return [g*xh + b for g, xh, b in zip(gamma, xhat, beta)]

def rms_norm(x, gamma=None, eps=1e-5):
    """RMSNorm: only divide by RMS — NO mean subtraction (7% faster)."""
    n = len(x)
    rms = math.sqrt(sum(v**2 for v in x) / n + eps)
    xhat = [v / rms for v in x]
    if gamma is None: gamma = [1.0]*n
    return [g*xh for g, xh in zip(gamma, xhat)]

3.5

Q, K, V — The Three Projections

Self-attention begins with three separate linear projections of each token's residual stream vector. The input x (shape: [T, d_model]) is multiplied by three learned weight matrices to produce queries, keys, and values: Q = xW_Q, K = xW_K, V = xW_V. These are not just different names for the same thing — they play fundamentally different roles in the computation.

Jay Alammar's filing cabinet analogy (2019) is the best intuition for these roles. Imagine you're searching a filing cabinet for information about the word "it" in the sentence "the animal was tired because it was hot":

  • Query (Q): A sticky note with your research topic — "what does 'it' refer to?" The query represents what the current token is looking for.
  • Key (K): The labels on each folder in the cabinet. Every other token advertises what it contains. "animal" might have a key that says "noun, entity, subject."
  • Value (V): The actual contents of the folder. When you match the query against a key, you retrieve the value — the representation that actually gets mixed into the output.

Elhage et al. (2021) formalise this more precisely with the QK circuit / OV circuit decomposition. Each attention head separates into two entirely independent computations: the QK circuit (W_Q^T W_K) determines the attention pattern — which positions attend to which — without caring what information moves. The OV circuit (W_O W_V) determines what information transfers when a position is attended to — without caring which positions are chosen. This independence is what enables mechanistic interpretability: you can study where a head attends separately from what it moves.

In GPT-2 small: d_model = 768, 12 heads, so d_head = 768/12 = 64 dimensions per head. Remarkably, d_head = 64 is constant across all GPT-2 variants: Medium has 16 heads (1024/16=64), Large has 20 heads (1280/20=64), XL has 25 heads (1600/25=64). The head dimension was fixed at 64 throughout the GPT-2 family — only the number of heads scales with model size.

Key insight: The three projections W_Q, W_K, W_V are learned, not fixed. Each head learns to project the residual stream into the subspace most useful for its particular computation. One head might learn to project into a "syntactic dependency" subspace; another into "semantic similarity." The projections are the mechanism by which each head specialises.

Scaled Dot-Product Attention

With Q, K, V in hand, attention is computed in five steps. For each query position, compute a score against every key, scale by √d_k, mask future positions, apply softmax, then sum the values weighted by the softmax output. The complete formula:

SCALED DOT-PRODUCT ATTENTION — VASWANI ET AL. 2017 Attention(Q, K, V) = softmax( QKT / √d_k ) · V Q ∈ ℝ^(T×d_k) | K ∈ ℝ^(T×d_k) | V ∈ ℝ^(T×d_v) | output ∈ ℝ^(T×d_v)

The √d_k scaling factor is not cosmetic — it's mathematically required. Consider two random vectors q and k, each of dimension d_k with elements drawn from N(0,1). Their dot product has variance d_k (the variance of a sum of d_k independent unit-variance terms). As d_k grows (d_k=64 in GPT-2), the raw dot products grow in magnitude. Large magnitude inputs push softmax into regions of near-zero gradient — the output approaches a one-hot distribution and backpropagation gets stuck. Dividing by √d_k restores the variance to 1.0 regardless of d_k.

Concretely: without scaling at d_k=64, a typical dot product is ±8. At d_k=512, it's ±22. The softmax of values in this range saturates immediately. With scaling: divide by √64 = 8, and the inputs to softmax are back to ±1 — the regime where gradients flow.

Alammar notes that GPT-2 uses −1 billion (not −∞) for masked positions. In float32, −1e9 is large enough that exp(−1e9) ≈ 0 numerically, but avoids the NaN that could arise from exact −∞ in edge cases. After softmax, these masked positions contribute exactly 0 to the output.

Key insight: The attention output at position i is a weighted average of value vectors: z_i = Σⱼ α_{ij} · v_j. Alammar's example: when processing "it" in "the animal was tired because it was hot," GPT-2's attention weights show 50% on "animal," 30% on "a," 19% on "it." The output is literally a mixture — 50% of what "animal" carries, 30% of "a," etc.

Multi-Head Attention

A single attention head is powerful but limited: it can only attend with one Q/K/V projection at a time. Multi-head attention runs H independent attention operations in parallel, each with its own learned W_Q, W_K, W_V. For GPT-2 small: H = 12 heads, d_head = 64 each. The results are concatenated and projected back to d_model=768 through a final W_O matrix.

The implementation detail that surprises most people: the heads are not run sequentially. Alammar calls the concatenation output a "Frankenstein's monster of hidden states" — you get one large matrix that's then projected through W_O. But more importantly, the computation is done via tensor reshaping, not 12 separate attention calls. The full [B, T, 768] Q/K/V matrix is reshaped to [B, 12, T, 64], making the head dimension explicit. All 12 heads compute attention simultaneously as a single batched operation — this is what makes MHA as fast as single-head attention on modern hardware.

Why multiple heads? Each head can specialise in a different relational structure. Head 3 might attend to syntactic dependencies; head 7 to semantic roles; head 11 to positional proximity. This emerges from training — no head is explicitly told what to learn. In GPT-2, Elhage et al. found heads with interpretable roles: one head consistently identifies what follows a "[X] is a" pattern (a "is-a" head); another tracks subject-verb agreement across long spans.

GQA and MQA: Modern LLMs reduce the memory cost of the KV cache by sharing keys and values across heads. Multi-Query Attention (MQA, Shazeer 2019) uses 1 K and 1 V but H queries. Grouped-Query Attention (GQA, Ainslie et al. 2023, used by LLaMA-2/3) uses G groups of heads sharing K and V, where G is between 1 (MQA) and H (full MHA). LLaMA-3 70B uses G=8 groups — 8 KV heads shared by 64 query heads.

Key insight: The W_O output projection is not optional decoration — it's where the 12 heads negotiate. Each head writes to a 64-dimensional subspace of the residual stream. W_O reads all 12 subspaces together (768 = 12 × 64) and writes a single 768-dim update back. The heads can't influence the residual stream independently; they must collectively pass through this bottleneck.

The Causal Mask

GPT-2 is an autoregressive model: it generates tokens left-to-right, and during generation, each new token can only see the tokens that came before it. The architecture must enforce this. The mechanism is the causal mask — a lower-triangular matrix of −1e9 values added to the attention scores before softmax, causing future-position attention weights to collapse to exactly zero.

The elegant part is what this enables during training. Rather than generating one token at a time (which would require T separate forward passes for a T-length sequence), the model processes the entire sequence in one pass. A batch of 512 sequences of length 1024 is processed simultaneously — and for each sequence, all T autoregressive predictions are made at once. The causal mask is what makes this possible: it prevents position 5 from "cheating" by attending to position 6's representation, even though position 6's representation is right there in the same forward pass.

Alammar describes the masking value as "−1 billion in GPT-2" — specifically -1e9 rather than float('-inf'). Both work in practice: exp(-1e9) ≈ 0 and exp(-∞) = 0 exactly. The advantage of −1e9 is numerical safety — exact infinity can produce NaN in edge cases when all scores in a row are −∞ (empty causal context), while −1e9 falls back gracefully to equal softmax weights.

Key insight: During training, GPT-2 uses batch size 512 and processes sequences of 1024 tokens. That means 512 × 1024 = 524,288 simultaneous autoregressive predictions per forward pass — all valid, all correctly masked. The causal mask converts a sequential generation problem into a parallel matrix computation, which is why transformers train efficiently on GPUs.

Flash Attention

Standard self-attention has an O(N²) memory problem. For a 4096-token sequence with d_head=128, the attention score matrix S = QK^T is shape [4096, 4096] — 67MB in FP16 per head, per layer. Materialising this in GPU HBM (high-bandwidth memory) and then immediately reading it back for softmax wastes enormous bandwidth. The computation is not FLOP-limited; it is memory-bandwidth-limited.

The A100 GPU has HBM bandwidth of ~2 TB/s and on-chip SRAM bandwidth of ~19 TB/s — approximately 10× faster. But SRAM is tiny: 192 KB per streaming multiprocessor. Flash Attention (Dao et al. 2022) exploits this asymmetry by tiling: instead of materialising the full N×N matrix, it processes Q, K, V in blocks small enough to fit in SRAM. The scaled dot-product attention is computed block by block, never writing the intermediate matrices to HBM.

FLASH ATTENTION IO COMPLEXITY — DAO ET AL. 2022 Standard: Θ(Nd + N²) HBM accesses
Flash: Θ(N²d²/M) HBM accesses
N = sequence length | d = head dimension | M = SRAM capacity per SM | Reduction ≈ N/(4d) at optimal tiling

The algorithmic trick is online softmax. Computing softmax across separated blocks is incorrect — the denominator spans the full row. Flash Attention maintains two running statistics: a row-wise maximum m and a row-wise sum of exponentials l. When a new block arrives, old outputs are rescaled by exp(m_old − m_new) before accumulating. The final result is mathematically exact — not an approximation.

Concrete IO at N=4096, d=128 on A100: standard attention accesses 134 MB of HBM; Flash Attention accesses 17 MB — an 8× reduction. Standard attention runs at 64 FLOP/byte (memory-bound, below the A100's ridge point of 156 FLOP/byte). Flash Attention runs at 506 FLOP/byte — fully compute-bound. This is why Flash Attention performs more FLOPs (recomputes attention during backward pass rather than storing it) yet runs 3× faster wall-clock.

def standard_attention_io(N, d):
    """HBM bytes: read Q,K,V + write/read S + write/read P + write O."""
    read_qkv   = 3 * N * d     # read Q, K, V
    write_s    = N * N         # write attention scores
    read_s     = N * N         # read scores for softmax
    write_p    = N * N         # write softmax probs
    read_pv    = N * N + N*d   # read P and V for output
    write_o    = N * d         # write output
    total_elems = read_qkv + write_s + read_s + write_p + read_pv + write_o
    return total_elems * dtype_bytes

def flash_attention_io(N, d, M):
    """HBM bytes with optimal tiling (Dao 2022, Theorem 1)."""
    Bc = min(math.ceil(M / (4*d)), d)   # block size for K,V
    Br = min(math.ceil(M / (4*d)), d)   # block size for Q
    passes = math.ceil(N / Bc)          # number of K,V block passes
    read_qkv   = 3 * N * d             # read Q,K,V once
    write_o    = N * d                  # write output once
    # statistics (m, l) per Q-block per K-block pass
    stats = 2 * math.ceil(N/Br) * passes
    total_elems = read_qkv + write_o + stats
    return total_elems * dtype_bytes

The Feedforward Sublayer

After attention, each token's representation passes through a two-layer feedforward network applied independently at each position. No token-to-token mixing here — just a per-position nonlinear function. The role is fundamentally different from attention: while attention decides where to look, the FFN decides what to think about what it saw.

The expansion is 4×: GPT-2 small expands 768 → 3,072, applies a nonlinearity, then projects back to 768. Alammar's explanation for why 4×: "That's just the size the original transformer rolled with. Model dimension was 512 and the inner layer was 2,048. This seems to give transformer models enough representational capacity." The 4× ratio is empirically stable across scales — GPT-3 (12,288-dim) uses 49,152 for the inner layer.

The nonlinearity is GELU (Gaussian Error Linear Unit). The exact formula:

GELU — HENDRYCKS & GIMPEL 2016 (APPROXIMATION USED IN GPT-2) GELU(x) = x · Φ(x) = x · 0.5 · (1 + tanh(√(2/π) · (x + 0.044715 · x³))) Probabilistic interpretation: GELU(x) = x · P(X ≤ x) where X ~ N(0,1). The neuron is stochastically gated — positive inputs pass, negative are smoothly suppressed.

The constant 0.044715 is empirically derived; the tanh approximation matches the exact GELU (via erf) to within 0.1%. The probabilistic framing is key: unlike ReLU which makes a hard binary decision (pass or kill), GELU attenuates negative activations proportionally. GELU has a minimum of −0.17 at x ≈ −0.5 — negative activations are never fully silenced, just reduced. This preserves gradient flow through negative pre-activations.

ReLU's dying problem: When a ReLU neuron's input is consistently negative (due to large negative bias), its gradient is exactly zero. The neuron never updates. In deep networks, 50.3% of ReLU neurons in FFN hidden layers have near-zero activation — they're dead. GELU reduces this to 5.1% by keeping negative activations alive with small nonzero values.

SwiGLU (Shazeer 2020), used by LLaMA/PaLM: SwiGLU(x, W, V) = SiLU(xW) ⊗ (xV) — a gated activation that multiplies two separate projections. To maintain the same parameter count, SwiGLU uses 8/3× expansion instead of 4× (LLaMA: 4096 → 11,008 instead of 16,384). Geva et al. (2021) showed that individual FFN neurons act as key-value memories — neurons fire for specific input patterns, and their value vectors point toward related output tokens.

Key insight: Geva et al. 2021 showed the FFN is a factual knowledge store. Individual neurons in the FFN respond to specific linguistic patterns ("the Eiffel Tower is located in…") and their value vectors contribute logit mass to factually related tokens. Attention is the retrieval mechanism; FFN is the knowledge base.

Residual Connections & Gradient Flow

Stack 96 matrix multiplications and backpropagate through them. The gradient — the correction signal for each weight — shrinks with every layer it passes through. After 48 layers of multiplying by matrices with singular values slightly less than 1, the gradient is near-zero. Weights stop updating. Training fails. This is the vanishing gradient problem that prevented deep learning from scaling before 2015.

The fix (He et al. 2016) is elegant: instead of computing output = Sublayer(x), compute output = x + Sublayer(LN(x)). The x term bypasses the sublayer. Differentiating through this addition gives:

RESIDUAL GRADIENT — THE +1 IDENTITY TERM ∂L/∂x = ∂L/∂y · (∂f/∂x + 1) The "+1" identity term ensures the gradient never fully vanishes — even if ∂f/∂x → 0, the gradient remains ∂L/∂y. Every early layer has a direct path back to the loss.

Karpathy describes the residual architecture precisely: "the residual pathway is the hero." The sublayers (attention, FFN) are learning corrections — small deltas to add to the stream. The stream itself flows largely unchanged from input embedding to logits. The weights near the input don't need the gradient to propagate through 95 layers — the residual connection gives them a shortcut.

Pre-norm (GPT-2) vs post-norm (Vaswani 2017): Xiong et al. 2020 proved theoretically that pre-norm transformers have more stable gradient norms at depth. With post-norm, the residual stream itself is normalised at each step — this interferes with the bypass. With pre-norm, the bypass carries the full-magnitude stream while the sublayer always sees a normalised input. At 96+ layers, pre-norm is the only option that trains reliably without careful learning rate tuning.

Key insight: The residual connection is the single most important architectural decision in the transformer. Without it, the original 6-layer Vaswani model trained only because 6 layers is relatively shallow. GPT-3's 96 layers would be impossible. The "+1" identity term is free — it requires no parameters, no computation, just an addition — and it makes the difference between training and not training at scale.

The Full Transformer Block

Stack the pieces: pre-norm LayerNorm, causal self-attention, residual add, pre-norm LayerNorm, feedforward, residual add. That's the entire transformer block — 4 operations, 2 sublayers. GPT-2 small runs this 12 times. GPT-3 runs it 96 times. The weights are different in every block, but the structure is identical. This regularity is what makes transformers so easy to scale: to double the model, add more blocks.

Blocks near the input tend to learn syntactic patterns — subject-verb agreement, dependency relations. Middle blocks build semantic representations — entity tracking, coreference. Late blocks focus on task-specific predictions — what word fits here given the full document context. This specialisation emerges from training; the architecture places no constraints on which block learns which role. Elhage et al. (2021) showed that two-layer attention-only models already learn nontrivial circuits — the induction head (§3.16) appears at layer 2 in models as small as 2 layers.

The residual stream is the central data structure through all 12 blocks. Think of it as a whiteboard: the embedding layer writes the starting position (token + position), each attention head reads and writes corrections, each FFN reads and writes corrections, and the final unembedding reads the result. The whiteboard grows richer with each block, accumulating the combined contributions of 12 rounds of token communication and per-token processing.

Key insight: The block is stateless. It has no memory between forward passes — no hidden state, no cell state, no recurrence. Everything the model "knows" about the current sequence is encoded in the residual stream vectors at this moment. The weights store the static "knowledge"; the residual stream stores the dynamic "working memory." This is why transformers can process prompts in parallel but generate one token at a time.

Rotary Position Embeddings (RoPE)

GPT-2's learned positional embeddings have a hard ceiling: they cannot extrapolate beyond the 1,024-position context they were trained on. Ask GPT-2 to process token 1,025 and the model has no positional representation for it — it either truncates or produces garbage. The root cause is that absolute position embeddings inject position as a fixed additive offset before any computation, divorcing the positional signal from the content it is supposed to qualify. RoPE (Rotary Position Embedding, Su et al. 2021, arXiv 2104.09864) solves this with a fundamentally different approach: rather than adding position to the token vector, it rotates the query and key vectors by an angle proportional to their position before the dot product.

The mathematical insight is elegant. Define the inner product between query at position m and key at position n. RoPE constructs rotation matrices R_m and R_n such that:

ROPE CORE IDENTITY — SU ET AL. 2021 ⟨R_m · q, R_n · k⟩ = ⟨q, R_{n−m} · k⟩ The absolute positions m and n disappear; only the relative offset (n − m) remains in the dot product. Relative position attention emerges automatically from absolute position rotations.

In practice, RoPE operates on pairs of dimensions. For a d-dimensional vector, it applies d/2 independent 2D rotations. Pair (i, i+1) is rotated by angle θ_i · pos, where θ_i = 10000^(−2i/d). This is the same geometric progression as sinusoidal PE — but applied multiplicatively as a rotation rather than additively as an offset. The rotation matrix for position pos across dimension pair i is:

ROPE ROTATION — DIMENSION PAIR i AT POSITION pos [cos(θ_i·pos) −sin(θ_i·pos)] [q_{2i} ] [sin(θ_i·pos) cos(θ_i·pos)] [q_{2i+1}] θ_i = 10000^(−2i/d_head). Each dimension pair rotates at a different frequency. Only Q and K are rotated — V is not. Applied before the QK dot product.

Three properties make RoPE the de facto standard positional encoding in modern LLMs (LLaMA 1/2/3, Mistral, Falcon, GPT-NeoX, PaLM 2, Gemma): (1) Relative position falls out naturally — no explicit relative attention bias needed. (2) Long-context extrapolation — by increasing the RoPE base from 10,000 to 500,000 (LLaMA-3), models trained on 8K tokens can be fine-tuned to 128K+ with minimal degradation. (3) Zero additional parameters — RoPE is a fixed function of position, not a lookup table. GPT-2's W_PE costs 786K parameters; RoPE costs zero.

Key insight: RoPE's extrapolation advantage comes from its frequency structure. At training context length 8K with base 10,000, the highest-frequency dimension completes thousands of full rotations — it has "seen" every phase. Extending to 16K just means more rotations of the same frequencies. With base 10,000, dimension 0 completes 8000/(2π) ≈ 1,273 cycles at 8K tokens. Dimension d/2−1 completes 0.0025 cycles. Increasing the base to 500,000 slows all dimensions by 50×, so even the slowest dimensions stay in their training range when extrapolating.

import math

def rope_rotate(vec, pos, base=10000):
    """Apply RoPE rotation to a vector at a given position.
    Operates on pairs of dimensions (d must be even)."""
    d = len(vec)                         # ↑ d must be even — we rotate in pairs
    assert d % 2 == 0
    out = list(vec)
    for i in range(d // 2):
        theta = base ** (-2 * i / d)     # ↑ Same frequency as sinusoidal PE
        angle = pos * theta              # ↑ Rotation angle = position × frequency
        cos_a, sin_a = math.cos(angle), math.sin(angle)
        x0, x1 = vec[2*i], vec[2*i+1]   # ↑ Take a pair of dimensions
        out[2*i]   =  x0 * cos_a - x1 * sin_a   # ↑ 2D rotation formula
        out[2*i+1] =  x0 * sin_a + x1 * cos_a   # ↑ standard rotation matrix
    return out

Grouped Query Attention (GQA) and the KV Cache

At inference time, the transformer generates tokens one at a time. Generating token t+1 requires computing attention over all t previous tokens. Without optimisation, this means re-computing the keys and values for every previous token on every generation step — a cost that grows quadratically with sequence length. The KV cache eliminates this redundancy: keys and values for all previous tokens are computed once and stored, then reused on every subsequent step. Only the new token's Q, K, V need to be computed fresh.

The memory cost is the problem. For a single sequence of length N, one layer of GPT-2 small (12 heads, d_head=64, FP16) stores: N × 12 × 64 × 2 bytes per matrix × 2 matrices (K and V) × 12 layers = N × 36,864 bytes. At N=1,024: 37.7 MB. For LLaMA-3 70B at N=8,192: 70 layers × 8 KV heads × 128-dim × 2 bytes × 2 × 8,192 = 14.7 GB — more than half the GPU's memory just for the KV cache of one sequence at full context.

Multi-Query Attention (MQA), introduced by Shazeer (2019), was the first solution: use H query heads but only 1 shared K and 1 shared V. The KV cache shrinks by a factor of H (×8 for an 8-head model). Quality drops modestly — queries can no longer attend from specialised perspectives since all share the same key/value projection. Grouped Query Attention (GQA), introduced by Ainslie et al. (2023, arXiv 2305.13245), finds the middle ground: G groups of query heads share one K/V pair each, where 1 ≤ G ≤ H. LLaMA-2 70B uses G=8 groups with H=64 query heads — 8 KV heads shared across 64 queries. KV cache is 8× smaller than MHA, and quality nearly matches MHA.

KV CACHE MEMORY — PER TOKEN, PER LAYER bytes = 2 × n_kv_heads × d_head × 2 bytes (FP16) Factor of 2: one for K, one for V. n_kv_heads = n_heads in MHA, 1 in MQA, n_heads/G in GQA. Scale by N×L for full cache.
Key insight: The KV cache is what makes LLM inference fast despite the autoregressive constraint. Without it, generating a 1,000-token response requires 1,000 full forward passes — each pass processing tokens 1 through t. With the cache, each pass processes only 1 new token: the generated token. The prefill phase (processing the input prompt) fills the cache; the decode phase (generating new tokens) consumes it one step at a time. This is why LLMs respond slowly to long prompts but generate tokens rapidly once they start.

Weight Initialisation & the Output Projection Scaling

Deep neural networks are exquisitely sensitive to how their weights are initialised. Too large: activations explode, gradients overflow, training diverges in step 1. Too small: activations shrink exponentially with depth, reaching zero before the first gradient update — effectively the vanishing activation problem at initialisation. The goal is to initialise such that the variance of activations remains constant as the signal propagates through the network. Xavier/Glorot initialisation (2010) achieves this for linear layers by drawing weights from N(0, 2/(fan_in + fan_out)).

nanoGPT uses a refined version: all weights from N(0, 0.02) — a conservative fixed standard deviation that keeps activations in a well-behaved range for the model sizes concerned. But there is one critical exception: the output projection layers (c_proj in both Attention and MLP). These are the layers that write directly back to the residual stream. Karpathy applies a modified initialisation to them:

OUTPUT PROJECTION INIT — NANOGPT (KARPATHY 2022) std = 0.02 / √(2 × n_layer) For GPT-2 small with 12 layers: std = 0.02 / √24 = 0.00408. The 2 accounts for both the attention output projection AND the MLP output projection — two additions per block.

The motivation comes from the residual stream analysis. Each of the L blocks adds its output directly to the stream. At initialisation, if each block adds a correction with standard deviation σ, the cumulative stream has standard deviation σ√L (sum of L independent random variables). For GPT-2-XL (48 layers), σ=0.02 gives stream std of 0.02√48 ≈ 0.14 — manageable. But for GPT-3 (96 layers), this grows to 0.02√96 ≈ 0.196. The /√(2L) scaling ensures that as the model deepens, the stream starts in a controlled state regardless of depth. Without it, the final LayerNorm before the LM head must compensate for increasingly large magnitudes — harder to optimise.

The full GPT-2 small parameter budget, counted precisely: embedding W_E (38.6M) + positional W_PE (0.79M) + 12 blocks × [LN1 (1.5K) + Attn W_QKV (1.77M) + Attn W_O (0.59M) + LN2 (1.5K) + MLP W_fc (2.36M) + MLP W_cp (2.36M)] + final LN (1.5K) + LM head (0 if tied to W_E) = 124M parameters total.

Key insight: The tied embedding weights (§3.16) mean the LM head is not an extra 38.6M parameters — it reuses W_E transposed. This is not a minor detail: in GPT-2 small, the embedding matrix is 31% of the total parameter count. Tying it saves ~38.6M parameters while improving coherence between input and output representations, because the model must learn embeddings that work both as input representations and as output logit weights simultaneously.

Induction Heads & In-Context Learning

Among the most striking results of mechanistic interpretability is the discovery that a specific circuit — the induction head — is responsible for in-context learning. Olsson et al. (2022, "In-context Learning and Induction Heads") showed that induction heads form in 2-layer models after approximately 2 billion training tokens, and that their formation coincides precisely with a sharp phase transition in in-context learning ability. Remove the induction heads, and the model loses its ability to learn from examples in the prompt.

The induction head circuit requires exactly two attention layers working together. Layer 1 — the prefix head: attends from each token to the token immediately before it, copying that token's identity into the current residual stream. Layer 2 — the induction head: reads the "what came before me" information written by layer 1 and attends back to wherever that pattern appeared previously in the sequence. The combined effect: if the sequence contains [A][B] ... [A], the induction head predicts [B] at the second [A] — it completes the pattern from prior context.

INDUCTION HEAD — CIRCUIT COMPOSITION (OLSSON ET AL. 2022) prefix_head writes: "token at position i−1 was [X]" induction_head reads: find position j where "token at j−1 was [X]" prediction: token at position j+1 This is the mechanism for few-shot learning: [cat → mat] [dog → ???] becomes an induction pattern where "what follows 'dog' in the prior context" is retrieved.

Induction heads are universal: they appear in every transformer trained on natural language, code, or arithmetic. They appear in models as small as 2 layers with 2 heads. They are not designed; they emerge spontaneously. Elhage et al. (2022) found that when induction heads form in a 2-layer model, the loss curves show a distinct "kink" — a sudden improvement that corresponds exactly to this circuit self-organising.

The broader implication is profound: in-context learning in a transformer is implemented by a small number of identifiable, composable circuits. The 12 attention heads in GPT-2 small are not all doing the same thing — specific heads implement specific functions. This is the program of mechanistic interpretability: reverse-engineer the algorithm the model has learned, head by head, circuit by circuit.

Key insight: The induction head is the simplest example of circuit composition across layers — Layer 1's output is literally the input to Layer 2's key computation. This means two-layer models can implement algorithms that one-layer models cannot: any computation that requires two sequential steps. GPT-2's 12 layers give it room for circuits composed of up to 12 sequential operations, enabling far richer internal algorithms than a shallow model.

The Generation Loop & Sampling Strategies

Everything in this chapter converges on a single function: generate. Given a prompt of T tokens, the model runs a forward pass producing logits[T−1] — a 50,257-dimensional vector of unnormalised log-probabilities over the next token. One token is sampled or selected. It is appended to the context. The forward pass runs again for position T+1. This continues until an end-of-sequence token is produced or a length limit is hit. The KV cache (§3.14) means only the new token needs a full forward pass; all previous keys and values are already computed.

The choice of how to select from the logit distribution has dramatic consequences for output quality. Five strategies, from simplest to most sophisticated:

  • Greedy (argmax): Always take the highest-probability token. Fast, deterministic, but repetitive — the model gets stuck in loops. Collapsed to 0 diversity.
  • Top-k sampling (Fan et al. 2018): Keep only the k highest-probability tokens, renormalise, and sample. k=50 is common. Prevents very low-probability tokens from occasionally polluting output.
  • Top-p (nucleus) sampling (Holtzman et al. 2019): Keep the smallest set of tokens whose probabilities sum to at least p. p=0.9 is typical. Adapts to the distribution — when the model is confident, only 2–3 tokens pass the threshold; when uncertain, 50+ tokens contribute.
  • Temperature scaling: Divide all logits by T before softmax. T<1 makes the distribution sharper (more deterministic). T>1 makes it flatter (more random). T=1 is the raw model probability. nanoGPT uses T=1.0 by default.
  • Min-p sampling (Nguyen 2023): Filter tokens below min_p × max_probability. Adaptive like nucleus but threshold scales with the peak probability — tighter filtering when confident.
TEMPERATURE-SCALED SAMPLING (INFERENCE) p_i = exp(z_i / T) / Σⱼ exp(zⱼ / T) T=0.0: argmax (greedy). T=1.0: raw model probabilities. T=2.0: near-uniform. GPT-4 default: T=1.0 + top_p=1.0 for chat, T=0 for code.

The prefill vs decode distinction is the most important operational fact about LLM inference. Prefill: all T prompt tokens are processed in one parallel forward pass — O(T²) attention computation, but highly parallelisable, runs fast on GPU. Decode: each new token is one sequential step — O(T) attention (thanks to KV cache), but sequential and cannot be parallelised across tokens. This asymmetry is why latency to first token (TTFT) scales with prompt length, but subsequent token generation speed (throughput) is nearly constant.

Key insight: The generation loop is the transformer's bottleneck in production. The KV cache trades memory for speed — it converts O(N²) recomputation into O(N) memory reads. But at scale (batch size 128, context 8K, 70B model), the KV cache alone requires 14.7 GB × 128 = 1.88 TB of memory — impossible on a single node. Production LLM serving systems (vLLM, TensorRT-LLM) use PagedAttention (Kwon et al. 2023) to page-manage the KV cache like virtual memory, enabling tens of thousands of concurrent requests on the same GPU cluster.

def softmax(x):
    mx = max(x); e = [math.exp(v-mx) for v in x]; s = sum(e); return [v/s for v in e]

def sample_categorical(probs, rand_val):
    """Sample from categorical distribution given a uniform random value."""
    cumsum = 0
    for i, p in enumerate(probs):
        cumsum += p
        if rand_val <= cumsum:
            return i
    return len(probs) - 1

def top_k_filter(logits, k):
    """Zero out all but top-k logits."""
    indexed = sorted(enumerate(logits), key=lambda x: -x[1])
    threshold = indexed[k-1][1]
    return [l if l >= threshold else -1e9 for l in logits]

def top_p_filter(logits, p):
    """Keep smallest set of tokens summing to >= p probability."""
    probs = softmax(logits)
    sorted_idx = sorted(range(N), key=lambda i: -probs[i])
    cum_p = 0
    keep = set()
    for i in sorted_idx:
        cum_p += probs[i]
        keep.add(i)
        if cum_p >= p: break
    return [logits[i] if i in keep else -1e9 for i in range(N)]
