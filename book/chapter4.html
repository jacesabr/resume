4.1

Layer Normalisation

Before any attention or feedforward operation, GPT-2 normalises the residual stream. This is not optional housekeeping — without it, training a 12-layer transformer with a constant learning rate would fail. Activations grow as they accumulate across residual additions; without normalisation the variance compounds multiplicatively through each layer, and gradients either explode or vanish before reaching the early layers.

The original Transformer used post-norm — applying LayerNorm after the residual addition. GPT-2 switched to pre-norm: normalise the input before passing it to attention or FFN, then add the residual. This small change dramatically improves training stability. The gradient of the residual bypass is always 1.0; pre-norm means the normalised input is always well-conditioned before the matrix multiplications, regardless of how the residual has accumulated.

The formula: given an input vector x of dimension d, subtract its mean and divide by its standard deviation, then apply learned scale γ and bias β. Both γ and β are initialised to 1 and 0 respectively, so initially LayerNorm is an identity function. Training adjusts them to allow different dimensions to have different typical magnitudes. The constant ε = 1e-5 prevents division by zero when all elements are identical.

LAYER NORMALISATION LN(x) = γ · (x − μ) / √(σ² + ε) + β μ = mean(x), σ² = variance(x), γ and β are learned per-dimension, ε = 1e-5
Key insight: Pre-norm (normalise before sublayer) is strictly better than post-norm for deep transformers. Pre-norm creates a "clean" gradient highway through the residual stream — the gradient of the shortcut path is exactly 1.0 at every layer.
layer_norm.py
import math

class LayerNorm:
    def __init__(self, emb_dim, eps=1e-5):
        self.eps = eps
        self.scale = [1.0] * emb_dim   # ↑ γ: learned scale, initialised to 1 (identity)
        self.shift = [0.0] * emb_dim   # ↑ β: learned bias, initialised to 0 (identity)

    def forward(self, x):
        # x is a list of floats (one token's embedding)
        n = len(x)
        mean = sum(x) / n                        # ↑ μ: mean across embedding dimension
        var  = sum((v - mean)**2 for v in x) / n  # ↑ σ²: variance across embedding dimension
        std  = math.sqrt(var + self.eps)          # ↑ add ε before sqrt to avoid division by 0
        norm = [(v - mean) / std for v in x]     # ↑ normalise: zero mean, unit variance
        return [self.scale[i] * norm[i] + self.shift[i]
                for i in range(n)]              # ↑ apply γ and β (affine transform)

4.2

The GELU Activation

Every position in the FeedForward sublayer passes through a nonlinear activation function. Without it, stacking linear layers is mathematically equivalent to one linear layer — the entire transformer would be a fancy linear regression. GPT-2 uses GELU (Gaussian Error Linear Unit) rather than the older ReLU.

ReLU is a hard gate: anything negative becomes exactly 0. This creates dead neurons — units that can never recover once their output becomes zero for a given input, because the gradient is also zero at those points. GELU softens the gate: rather than hard-clipping negative values, it multiplies each input by the probability that it is positive under a Gaussian distribution. Negative inputs are heavily suppressed but not completely silenced. The function is smooth and differentiable everywhere — gradients flow even for negative values, which preserves learning signal through deep stacks.

The exact formula involves the Gaussian CDF Φ, approximated with tanh for fast computation. The approximation error is negligible in practice. A concrete comparison: GELU(-1) ≈ -0.159, while ReLU(-1) = 0. That small negative output means the neuron still participates, weakly, and can recover if gradients shift.

GELU ACTIVATION GELU(x) = x · Φ(x) ≈ 0.5x · (1 + tanh(√(2/π) · (x + 0.044715x³))) Φ(x) = Gaussian CDF = P(X ≤ x) where X ~ N(0,1). The tanh form is GPT-2's fast approximation.

ReLU(x)

Hard gate: max(0, x). Zero and gradient-zero for all negative inputs. Simple but causes dead neurons. Used in earlier models (GPT-1 used GELU but many contemporaries used ReLU).

f(x) = max(0, x)

GELU(x)

Smooth probabilistic gate. Negative values suppressed but not zeroed — small gradient allows recovery. Matches the output distribution of stochastic depth regularisation.

f(x) ≈ 0.5x(1 + tanh(0.7978(x + 0.044715x³)))
Key insight: GELU is the standard activation in every major language model: BERT, GPT-2, GPT-3, LLaMA, Mistral. The smooth gate allows gradients to flow even for slightly negative pre-activations, enabling learning signal to penetrate dozens of layers.
gelu.py
import math

def gelu(x):
    # GPT-2's exact tanh approximation of GELU
    inner = math.sqrt(2.0 / math.pi) * (x + 0.044715 * x**3)  # ↑ polynomial inside tanh
    return 0.5 * x * (1.0 + math.tanh(inner))               # ↑ x times probability x is positive

def relu(x):
    return max(0.0, x)   # ↑ hard gate: negative inputs silenced completely

4.3

The FeedForward Sublayer

Each transformer block contains two sublayers: multi-head attention and a position-wise feedforward network. The feedforward network (FFN) is architecturally simple — two linear projections with a nonlinearity between them — but it contains the majority of the parameters in the entire model. In GPT-2, 67% of parameters live in FFN layers.

The design is: Linear(d_model → 4·d_model) → GELU → Linear(4·d_model → d_model). The expansion factor of 4× is a convention from the original "Attention Is All You Need" paper (Vaswani et al. 2017) and has been retained essentially unchanged in every transformer since. Recent work suggests 8/3× (used in LLaMA's SwiGLU variant) is slightly better, but 4× remains standard. The intermediate dimension is often called d_ff or d_inner.

Why expand then contract? The expansion creates a higher-dimensional space where the model can perform nonlinear feature interactions that attention cannot. Attention routes information between positions; FFN transforms the information at each position independently. Mechanistic interpretability research (Elhage et al. 2021) suggests FFN layers act like key-value memory stores — patterns in the first weight matrix "key" into factual associations stored in the second matrix's "values."

Key insight: For GPT-2 124M with d_model=768: each FFN layer has 2 × 768 × 3072 = 4,718,592 parameters. With 12 blocks, FFN alone accounts for ~56.6M parameters. The attention Q,K,V,O projections add only ~28.3M. The FFN is where knowledge is stored; attention is where it's routed.
feedforward.py
class FeedForward:
    def __init__(self, emb_dim):
        d_ff = 4 * emb_dim  # ↑ 4× expansion: convention from Vaswani et al. 2017
        # In PyTorch: nn.Linear(emb_dim, d_ff) and nn.Linear(d_ff, emb_dim)
        self.d_model = emb_dim
        self.d_ff = d_ff
        # Weights W1 shape: (emb_dim, d_ff), W2 shape: (d_ff, emb_dim)

    def param_count(self):
        # Each Linear has weight + bias
        w1 = self.d_model * self.d_ff   # ↑ up-projection: d_model → 4·d_model
        b1 = self.d_ff
        w2 = self.d_ff * self.d_model   # ↑ down-projection: 4·d_model → d_model
        b2 = self.d_model
        return w1 + b1 + w2 + b2       # ↑ total params in this FFN sublayer

4.4

Shortcut Connections

A transformer without residual connections would be essentially untrainable beyond 4–5 layers. The reason is the vanishing gradient problem: during backpropagation, the gradient must pass through every matrix multiplication. Each multiplication can shrink or grow the gradient — and over 12 layers, the effect compounds. By the time gradients reach the first layer, they can be 10⁻¹² of their original magnitude, carrying essentially no useful information.

Residual connections, introduced by He et al. in 2016 for image recognition (ResNets), solve this with a bypass: instead of transforming x to F(x), each block computes x + F(x). The gradient of the bypass is always exactly 1.0 — it passes through unchanged. The model therefore always has a gradient highway from output to input, regardless of depth. Each block only needs to learn a "correction" δ to the residual stream, not a complete new representation. This means early layers see gradients comparable in magnitude to the last layer.

GPT-2's forward pass for one block: x₁ = x₀ + Attn(LN(x₀)), then x₂ = x₁ + FFN(LN(x₁)). The two additions are the residual connections. Note the pre-norm pattern: LayerNorm is applied before the sublayer, and the raw (un-normalised) residual is added back. This preserves the identity function in the residual stream.

Key insight: Residual connections are why transformers can be trained to 96 layers (GPT-3) or 200+ layers (modern MoEs) at all. Without them, gradient signal would vanish in the first 10 layers. The shortcut turns deep networks into ensembles of shallower networks — if an early block learns nothing useful, its contribution approaches zero and the bypass handles the signal.
residual.py
# Residual connection pattern — appears twice in every transformer block

def transformer_block_forward(x, attn, ffn, ln1, ln2):
    # First sublayer: pre-norm + attention + residual
    x = x + attn(ln1(x))  # ↑ add residual: gradient highway bypasses attn

    # Second sublayer: pre-norm + feedforward + residual
    x = x + ffn(ln2(x))   # ↑ add residual: gradient highway bypasses FFN

    return x  # ↑ residual stream enriched with corrections from both sublayers

# Mathematical intuition:
# If Attn(LN(x)) ≈ 0, then x + 0 = x — block passes signal through unchanged.
# The block only needs to learn what to ADD, not what to output from scratch.

▶ Cell 4 — Demonstrate vanishing gradient with and without residuals 
import math, random
random.seed(42)

# Simulate gradient flow through N layers
# Without residual: grad multiplied by weight norms at each layer
# With residual: grad is sum (bypass always 1.0)

def simulate_grad(n_layers, weight_scale=0.9, use_residual=True):
    grad = 1.0
    for _ in range(n_layers):
        layer_grad = weight_scale  # simplified: grad shrinks by weight_scale each layer
        if use_residual:
            grad = grad * 1.0 + grad * layer_grad  # bypass (1.0) + sublayer
            grad /= 2  # normalise for fair comparison
        else:
            grad = grad * layer_grad
    return grad

print(f"{'Layers':<10}  {'No Residual':>14}  {'With Residual':>14}")
print("-" * 42)
for n in [1, 4, 8, 12, 24, 48, 96]:
    no_res = simulate_grad(n, use_residual=False)
    res    = simulate_grad(n, use_residual=True)
    print(f"{n:<10}  {no_res:>14.6f}  {res:>14.6f}")
# With residual, gradient stays above 0.5 even at 96 layers.
4.5

The Transformer Block

A transformer block combines everything from chapters 3 and 4 into one reusable unit: two LayerNorms, multi-head causal attention, a feedforward sublayer, two residual connections, and dropout for regularisation. This block is the atom of the GPT architecture — the full model is 12 identical copies of it stacked vertically, each with independently learned weights.

The data path through one block: the residual stream enters with shape (batch, seq_len, d_model). It passes through LayerNorm₁, then multi-head attention — which mixes information across positions. The attention output is added back to the original residual (first shortcut). The result passes through LayerNorm₂, then the FeedForward network — which transforms each position independently. The FFN output is added back again (second shortcut). The block's output has the same shape as its input, making blocks trivially stackable.

For GPT-2 small, one block's parameters: 2 LayerNorms (2×768×2 = 3,072), attention (768×3×768 for QKV + 768×768 for O = 2,359,296), FFN (768×3072×2 = 4,718,592, plus biases). Total per block: ~7,086,336. Times 12 blocks: ~85M of GPT-2's 124M total parameters.

Key insight: The block is stateless with respect to position — it sees all positions equally in the residual stream. Position information enters only once (via positional embeddings at the very start) and propagates through all blocks via the residual stream. Each block refines the representation, not the position encoding.
transformer_block.py
# Full transformer block (PyTorch-style pseudocode)
class TransformerBlock:
    def __init__(self, emb_dim, n_heads, drop_rate=0.1):
        self.ln1  = LayerNorm(emb_dim)         # ↑ pre-norm before attention
        self.attn = MultiHeadAttention(emb_dim, n_heads)  # ↑ causal self-attention
        self.ln2  = LayerNorm(emb_dim)         # ↑ pre-norm before feedforward
        self.ff   = FeedForward(emb_dim)       # ↑ position-wise FFN (4× expansion)
        self.drop = drop_rate                  # ↑ dropout rate (0.1 for GPT-2)

    def forward(self, x):
        # First sublayer: attention with pre-norm and residual
        x = x + dropout(self.attn.forward(self.ln1.forward(x)), self.drop)
        # ↑ add residual: x unchanged if attn output is 0

        # Second sublayer: FFN with pre-norm and residual
        x = x + dropout(self.ff.forward(self.ln2.forward(x)), self.drop)
        # ↑ add residual: each position refined independently by FFN
        return x

def block_params(emb_dim, n_heads):
    d_ff = 4 * emb_dim
    d_head = emb_dim // n_heads

    ln1    = emb_dim * 2                  # scale + shift
    ln2    = emb_dim * 2
    qkv    = 3 * emb_dim * emb_dim + 3 * emb_dim  # W_QKV + bias

        "Attention (QKV+O)": qkv + out_pr,
        "FeedForward": ff1 + ff2,

4.6

Stacking Blocks

The GPT architecture stacks N identical transformer blocks, each with its own independent weights. GPT-2 small uses 12 blocks, medium uses 24, large uses 36, XL uses 48. The residual stream — a tensor of shape (batch_size, seq_len, d_model) — flows through each block in sequence. Each block reads from the stream, computes a correction, and writes it back via the residual addition.

The blocks are not actually identical in function even though they share the same architecture. Each has randomly initialised, independently learned weights. Early blocks (1–4) tend to learn syntactic patterns: part-of-speech, named entity boundaries, punctuation structure. Middle blocks (5–8) tend to encode semantic associations and coreference. Late blocks (9–12) tend to specialise in task-specific patterns and the probability distribution over next tokens. This hierarchy emerges from training, not design.

In PyTorch, the standard idiom is nn.ModuleList — not a Python list. The difference matters: nn.ModuleList registers all contained modules as sub-modules of the parent, so their parameters appear in model.parameters() for the optimizer. A Python list of modules would be invisible to PyTorch's parameter tracking, and none of those parameters would be updated during training.

Key insight: Doubling the number of blocks roughly doubles performance (measured by perplexity loss) for the same compute budget — up to a point. This is the depth scaling law. GPT-3's 96 layers and LLaMA-3's 80 layers exploit this. But width (d_model) also matters: GPT-3 chose d_model=12,288, which is 16× GPT-2's 768.
block_stack.py
# Block stack in PyTorch-style pseudocode
class BlockStack:
    def __init__(self, n_layers, emb_dim, n_heads):
        # ModuleList: all blocks tracked as named submodules
        self.blocks = [
            TransformerBlock(emb_dim, n_heads)
            for _ in range(n_layers)
        ]                                         # ↑ n_layers independent blocks, each ~7M params

    def forward(self, x):
        for block in self.blocks:
            x = block.forward(x)              # ↑ each block enriches the residual stream
        return x                               # ↑ final residual stream after all 12 blocks

# GPT-2 small: 12 blocks × 7,086,336 params = 85,036,032 block parameters
# These are 68% of GPT-2's total 124M parameters

def gpt2_params(n_layers, d_model, n_heads, vocab=50257, ctx=1024):
    d_ff = 4 * d_model
    emb = vocab * d_model + ctx * d_model          # token + position embeddings
    per_block = (2*2*d_model +                     # 2 LayerNorms
                 3*d_model*d_model + 3*d_model +   # QKV proj + bias
                 d_model*d_model + d_model +        # O proj + bias
                 d_model*d_ff + d_ff +              # FFN W1 + b1
                 d_ff*d_model + d_model)            # FFN W2 + b2
    total = emb + n_layers * per_block + 2*d_model # + final LN (no lm_head: tied)
    return total

4.7

The Full GPT Model

Assembling the complete model is anticlimactic after the component work — the pieces click together without surprises. The forward pass is: token embedding lookup → position embedding lookup → add both → dropout → 12 transformer blocks → final LayerNorm → LM head (linear to vocab size). The input is a sequence of token IDs; the output is logits over the 50,257-token vocabulary for every position.

Token embeddings convert integer token IDs into d_model-dimensional vectors. Each ID is a row index into a (vocab_size × d_model) weight matrix. This is a lookup table, not a multiplication — fast and exactly equivalent to one-hot encoding × weight matrix, but stored as a direct table. Position embeddings are identical in structure: a (context_len × d_model) matrix, where each row encodes one position. GPT-2 uses learned position embeddings, not the sinusoidal ones from the original Transformer.

The final LayerNorm (after all blocks, before the LM head) is present in GPT-2 but absent from the original Transformer. It stabilises the residual stream before the final projection. Without it, the raw residual stream variance can be large, making the pre-softmax logits unstable. The LM head is a Linear(d_model, vocab_size) with no bias — its weight matrix is tied to the token embedding matrix (see §4.8).

gpt_model.py
# GPT-2 architecture — configuration for 124M ("small")
GPT_CONFIG = {
    "vocab_size":  50257,  # ↑ GPT-2 BPE vocabulary: 50,000 merges + 256 byte tokens + <|endoftext|>
    "context_len": 1024,   # ↑ maximum sequence length (context window)
    "emb_dim":     768,    # ↑ residual stream dimensionality (d_model)
    "n_heads":     12,     # ↑ attention heads: each sees 768/12 = 64 dims
    "n_layers":    12,     # ↑ transformer blocks stacked vertically
    "drop_rate":   0.1,   # ↑ dropout on embeddings, attention, FFN outputs
}

class GPTModel:
    def __init__(self, cfg):
        self.tok_emb = Embedding(cfg["vocab_size"],  cfg["emb_dim"])  # ↑ token ID → vector
        self.pos_emb = Embedding(cfg["context_len"], cfg["emb_dim"])  # ↑ position → vector
        self.drop    = Dropout(cfg["drop_rate"])
        self.blocks  = [
            TransformerBlock(cfg["emb_dim"], cfg["n_heads"])
            for _ in range(cfg["n_layers"])   # ↑ n_layers identical-architecture blocks
        ]
        self.final_norm = LayerNorm(cfg["emb_dim"])  # ↑ final norm before LM head
        self.lm_head    = Linear(cfg["emb_dim"], cfg["vocab_size"], bias=False)  # ↑ project to vocab
        # Weight tying: lm_head.weight = tok_emb.weight (see §4.8)

    def forward(self, token_ids):
        positions = range(len(token_ids[0]))     # ↑ [0, 1, 2, ..., seq_len-1]
        x = self.tok_emb.forward(token_ids) + self.pos_emb.forward(positions)
        # ↑ token meaning + position: both (batch, seq, 768), summed elementwise
        x = self.drop.forward(x)
        for block in self.blocks:
            x = block.forward(x)               # ↑ residual stream enriched 12 times
        x = self.final_norm.forward(x)
        logits = self.lm_head.forward(x)        # ↑ (batch, seq, 50257): raw scores per token
        return logits

Weight Tying

GPT-2's LM head — the final Linear layer that maps from d_model to vocab_size — shares its weight matrix with the token embedding layer. In PyTorch: lm_head.weight = tok_emb.weight. This single line eliminates 38.6 million parameters (50,257 × 768 × 4 bytes = 147 MB) from the model. It's not just a memory optimisation — it improves the model by enforcing a geometric consistency.

The intuition: if "king" is embedded as a vector e_king, then predicting "king" as the next token should give a high score when the current representation is close to e_king. The score is computed as h · W_lm^T, where h is the hidden state. If W_lm = W_emb (tied), then the score for token i is h · e_i — the dot product of the hidden state with the token embedding. Tokens whose meanings are close in embedding space get similar prediction scores, which is exactly what you want. Separate matrices would need to learn this consistency independently.

Weight tying was introduced by Press and Wolf (2017) and is now standard in virtually every language model: BERT, GPT-2, GPT-3, LLaMA, Mistral, Llama-3. It improves perplexity by ~1–2 points on standard benchmarks while halving the embedding memory footprint. During training, gradients flow into the shared matrix from both the embedding and the LM head — the embedding gets training signal from two directions simultaneously.

Counting Parameters

GPT-2 "small" has 124 million parameters. That number is cited everywhere, but rarely explained. Here is the exact breakdown. Understanding it is important because it reveals where the model's capacity is actually located — and why scaling in certain directions (depth vs. width) has different effects on capability.

The largest single component is the token embedding matrix: 50,257 × 768 = 38,597,376 parameters. This is 31% of the model. The position embedding adds 1,024 × 768 = 786,432. Then come 12 transformer blocks, each with 7,086,336 parameters (see §4.5): 12 × 7,086,336 = 85,036,032 total. A final LayerNorm contributes 1,536 parameters. The LM head shares weights with the token embedding: 0 extra. Total: ~124,421,376 — close to 124M.

A useful mental model: each transformer block adds about 7M parameters and costs roughly one "unit" of model capacity. Doubling the number of blocks doubles the block parameter count and roughly halves the perplexity on a standard corpus. Doubling d_model is more expensive: all embedding, attention, and FFN dimensions scale quadratically in block parameters (attention is O(d²), FFN is O(d²)), but provides richer representations at each position.

def count_gpt2_params(vocab=50257, ctx=1024, d=768, n_heads=12, n_layers=12):

4.10

Generating Text

Text generation is the forward pass used as a loop. Given an input sequence of token IDs, run a forward pass to get logits for the next token, sample or select one token, append it to the sequence, and repeat. Each iteration extends the context by one token. This is the autoregressive generation loop that underlies every chatbot, code completion engine, and creative writing tool.

The simplest strategy is greedy decoding: at every step, pick the token with the highest logit (argmax). This is fast and deterministic but produces repetitive, low-diversity text — because the highest-probability continuation is often a common phrase that repeats the most likely continuation again and again. Temperature sampling divides logits by a temperature T before softmax. T = 1 gives the raw model distribution; T < 1 sharpens it (more confident, less diverse); T > 1 flattens it (more random). Top-k restricts sampling to only the k highest-probability tokens, preventing extremely unlikely continuations from occasionally being selected.

The generation loop must respect the context length. GPT-2's context window is 1,024 tokens — once the sequence exceeds that, the oldest tokens must be discarded (a naive sliding window) or the model must truncate. Modern long-context models (Llama-3 with 128K context) extend this with architectural changes (RoPE, GQA), but the generation loop structure is identical.

1

Encode prompt

Convert "The sky is" → token IDs [464, 6766, 318] using the tokenizer.

2

Forward pass

Run GPTModel(token_ids) → logits shape (1, seq_len, 50257). Take last position: logits[-1].

3

Sample next token

Apply temperature, optionally top-k, then softmax → sample one token ID.

4

Append and repeat

Append sampled token to sequence. Go to step 2. Stop at max_new_tokens or <|endoftext|>.

5

Decode

Convert the full token ID sequence back to text using the tokenizer.

Key insight: The model runs a complete forward pass on the entire context at every generation step. This means generating 100 tokens requires 100 forward passes on contexts of length 1, 2, …, 100. The KV cache (Chapter 3, §3.11) avoids recomputing attention for already-seen tokens — cutting generation from O(n²) to O(n) per token.
generate.py
import math, random

def generate(model, token_ids, max_new=50, temperature=1.0, top_k=50):
    for _ in range(max_new):
        # Truncate to model's context length if needed
        ctx = token_ids[-model.context_len:]              # ↑ keep only last context_len tokens

        logits = model.forward(ctx)[-1]                 # ↑ last position's logits: shape (vocab,)

        # Apply temperature: scale logits before softmax
        logits = [l / temperature for l in logits]       # ↑ T<1 sharpens, T>1 flattens distribution

        # Top-k: zero out all but top_k logits
        if top_k is not None:
            sorted_idx = sorted(range(len(logits)), key=lambda i: -logits[i])
            cutoff = logits[sorted_idx[top_k]]            # ↑ value at position k
            logits = [l if l >= cutoff else -float('inf') for l in logits]  # ↑ mask the rest

        # Softmax → probability distribution
        m = max(logits)
        exps = [math.exp(l - m) for l in logits]
        s = sum(exps)
        probs = [e / s for e in exps]                  # ↑ sum-to-1 distribution over vocabulary

        # Sample one token from the distribution
        r = random.random()
        cumulative, next_token = 0.0, 0
        for i, p in enumerate(probs):
            cumulative += p
            if r <= cumulative:
                next_token = i; break               # ↑ inverse CDF sampling

        token_ids.append(next_token)                     # ↑ extend context with new token

        if next_token == model.eos_token_id:
            break                                        # ↑ stop at end-of-text token

    return token_ids

import math, random
random.seed(0)

# Toy vocabulary and "model" logits
vocab = ["the", "a", "cat", "sat", "dog", "jumped"]
logits = [3.2, 2.1, 1.8, 1.2, 0.9, 0.4]  # raw model logits for next token

def softmax_temp(logits, T=1.0):
    scaled = [l/T for l in logits]
    m = max(scaled)
    exps = [math.exp(l-m) for l in scaled]
    s = sum(exps)
    return [e/s for e in exps]

def sample(probs):
    r = random.random()
    acc = 0.0
    for i, p in enumerate(probs):
        acc += p
        if r <= acc:
            return i
    return len(probs) - 1
