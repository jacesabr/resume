What Is a Large Language Model?

A large language model is, at its core, a function that maps a sequence of tokens to a probability distribution over the next token. That deceptively simple objective — predict what comes next — when applied to enough data with enough parameters, gives rise to systems that translate, reason, write code, and hold coherent conversations.

The "large" in LLM refers to both parameter count (billions) and training data (trillions of tokens). Neither alone is sufficient: scale interacts. A 7B-parameter model trained on 1T tokens outperforms a 70B model trained on 100B tokens on many tasks, as the Chinchilla scaling laws (§1.9) formalise.

The autoregressive loop. At inference time, the model is called repeatedly. The output token is appended to the input, and the model runs again — until an end-of-sequence token is produced or a length limit is hit. Every output token is a one-way commitment: there is no backtracking.

The choice of how to select the next token from that probability distribution matters enormously. Greedy decoding (always take the highest-probability token) is fast but produces repetitive text. Beam search, long the gold standard in machine translation, generates text that feels "hollow and artificial," as Holtzman et al. (2019) demonstrated. Nucleus (top-p) sampling — first truncate the distribution to the smallest set of tokens whose probabilities sum to at least p, then sample — empirically produces the most coherent and diverse text.

import numpy as np

def softmax(logits, temperature=1.0):
    # Temperature > 1 → flatter dist (more random)
    # Temperature < 1 → sharper dist (more greedy)
    logits = np.array(logits) / temperature
    e = np.exp(logits - logits.max())  # numerically stable
    return e / e.sum()

def top_p_sample(logits, p=0.9, temperature=1.0):
    probs = softmax(logits, temperature)
    sorted_idx = np.argsort(probs)[::-1]
    cumulative = np.cumsum(probs[sorted_idx])
    # Keep only tokens in the nucleus
    cutoff = np.searchsorted(cumulative, p) + 1
    nucleus_idx = sorted_idx[:cutoff]
    nucleus_probs = probs[nucleus_idx]
    nucleus_probs /= nucleus_probs.sum()
    return np.random.choice(nucleus_idx, p=nucleus_probs)

Tokenisation

Models don't see characters or words — they see tokens. A token is a subword unit discovered by a compression algorithm. The dominant approach is Byte-Pair Encoding (BPE), introduced by Sennrich et al. (2016). BPE starts with individual bytes and iteratively merges the most frequent adjacent pair until a target vocabulary size is reached. GPT-4 uses a vocabulary of 100,257 tokens (the cl100k_base encoding).

BPE has some surprising failure modes. The string "SolidGoldMagikarp" was a Reddit username that appeared so rarely it effectively became a single token, causing GPT-3 to output random text when prompted with it. Numbers tokenise inconsistently: 1234 might be one token, but 12345 might be three, which is why LLMs struggle with arithmetic. Languages with richer morphology (Turkish, Finnish) have much higher token fertility — more tokens per word — making them effectively "more expensive" to process.

WordPiece vs BPE vs SentencePiece. BERT uses WordPiece, which maximises the likelihood of the training corpus rather than frequency. SentencePiece (used by LLaMA, T5) operates on raw bytes and avoids language-specific pre-tokenisation, making it truly language-agnostic.

Embeddings

Each token is mapped to a dense vector of floating-point numbers — its embedding. These vectors are learned during training. The geometry of the resulting space encodes semantic relationships: similar words cluster together, and vector arithmetic often mirrors conceptual arithmetic. The famous king − man + woman ≈ queen result (Mikolov et al., 2013) is a property that emerges automatically — nobody hardcoded it.

Modern LLMs use embedding dimensions of 4,096 (LLaMA-3 8B) to 12,288 (GPT-3 175B). The embedding table alone can be large: 100K tokens × 4,096 dims × 2 bytes (BF16) = 800MB, which is why tying input and output embeddings (using the same matrix for both) saves substantial memory.

The antonym paradox. "hot" and "cold" have high cosine similarity despite being opposites, because they appear in identical contexts ("it was very ___"). Embedding similarity captures distributional similarity, not conceptual similarity. This is a fundamental limitation worth understanding.

Transformer Architecture

The transformer (Vaswani et al., 2017) stacks L identical blocks, each containing two sublayers: multi-head self-attention and a position-wise feedforward network. Each sublayer is wrapped in a residual connection and followed by layer normalisation. The residual streams are the key architectural insight: gradients flow directly from output to input through additive skip connections, making training stable even at 100+ layers.

Modern decoder-only LLMs (GPT family, LLaMA, Mistral) use Pre-Norm (LayerNorm before the sublayer) rather than the original Post-Norm, which further stabilises training. Llama 3 uses RMSNorm instead of LayerNorm — it omits the mean-centering step, saving computation with negligible quality difference.

def count_params(d, d_ff, L, H, kv_heads, V, rope=True):
    """Approximate parameter count for a decoder-only LLM."""
    # Embedding
    embed = V * d

    # Per-layer attention: Q, K, V, O projections
    d_h = d // H       # head dimension
    q_proj  = d * d              # Q: d→d
    k_proj  = d * (kv_heads * d_h)  # K: d→kv_heads*d_h
    v_proj  = d * (kv_heads * d_h)  # V: same
    o_proj  = d * d              # O: d→d
    attn = q_proj + k_proj + v_proj + o_proj

    # Per-layer FFN (SwiGLU uses gate+up+down, 2/3 intermediate)
    d_int = int(d_ff)   # intermediate dim (already 2/3 * 4d for SwiGLU)
    ffn = d * d_int * 2 + d_int * d  # gate, up, down

    # Norms (RMSNorm: just one scale per dim, no bias)
    norm = d * 2 * L   # 2 norms per layer

    # LM head (often tied to embeddings, so not counted separately)
    total = embed + L * (attn + ffn) + norm
    return total

Self-Attention

Self-attention computes, for each token, a weighted average of all other token representations. The weights come from the similarity between that token's query vector and every other token's key vector. The value vectors are then averaged using these weights. The operation is differentiable end-to-end — the model learns which relationships to pay attention to purely from data.

import numpy as np

def softmax(x, axis=-1):
    e = np.exp(x - x.max(axis=axis, keepdims=True))
    return e / e.sum(axis=axis, keepdims=True)

def attention(Q, K, V, mask=None):
    d_k = Q.shape[-1]
    scores = Q @ K.T / np.sqrt(d_k)   # (T, T)
    if mask is not None:
        scores = np.where(mask, scores, -1e9)
    weights = softmax(scores)
    return weights @ V, weights

Multi-Head Attention & GQA

Running attention once gives the model one "perspective." Multi-head attention (MHA) runs H separate attention operations in parallel, each with smaller head dimension d_k = d_model/H. Each head learns to attend to different relationship types — one might track subject-verb agreement, another coreference chains. The outputs are concatenated and projected back to d_model.

For large models, MHA's memory cost becomes prohibitive during inference: storing K/V pairs for all H heads in the KV cache. Multi-Query Attention (MQA) (Shazeer, 2019) uses a single K and V head shared across all Q heads — reducing KV cache size by 8–32× at the cost of some model quality. Grouped-Query Attention (GQA) (Ainslie et al., 2023) is the sweet spot: G groups of Q heads share one K/V head each. LLaMA-3 uses 32 Q heads and 8 KV heads (4 Q heads per KV group).

Positional Encoding

Attention itself is permutation-invariant — if you shuffle all tokens, the attention weights change but the computation is otherwise identical. Position must be injected explicitly. The original transformer used fixed sinusoidal encoding: PE(pos, 2i) = sin(pos/10000^(2i/d)), PE(pos, 2i+1) = cos(…). The key property is that the relative offset (pos+k) can be expressed as a linear function of (pos), which the model can learn.

Rotary Position Embedding (RoPE) (Su et al., 2024) is the current standard, used in LLaMA, Mistral, and GPT-NeoX. RoPE encodes position by rotating Q and K vectors by an angle proportional to position. When Q·K is computed, the dot product naturally depends only on the relative distance (m−n), not absolute positions. This allows zero-shot generalisation to unseen sequence lengths and forms the foundation for extending context windows.

def rope_rotate(x, pos, base=10000):
    """Apply RoPE to a single (d,) vector at given position."""
    d = x.shape[0]
    half = d // 2

import numpy as np

def sinusoidal_pe(T, d, base=10000):
    pe = np.zeros((T, d))
    pos = np.arange(T)[:, None]
    div = np.exp(np.arange(0, d, 2) * -(np.log(base) / d))
    pe[:, 0::2] = np.sin(pos * div)
    pe[:, 1::2] = np.cos(pos * div[:d//2])
    return pe

The Feedforward Sublayer

After attention routes information between token positions, the feedforward network (FFN) processes each token independently and in parallel. The standard FFN applies two linear layers with a nonlinearity: FFN(x) = W₂ · GELU(W₁ · x). The intermediate dimension is 4× the model dimension — for LLaMA-3 8B that's 4,096 → 16,384 → 4,096.

Geva et al. (2021) made a striking discovery: FFN layers act as key-value memories. The first layer projects the input to a high-dimensional space of "keys," GELU activation selects which keys are active, and the second layer reads out the corresponding "values." Specific factual knowledge ("Paris is the capital of France") can be traced to specific neurons in specific FFN layers — lower layers store syntactic patterns, upper layers store semantic facts.

SwiGLU (Shazeer, 2020) replaces GELU with a gated variant: FFN(x) = W₂ · (SiLU(W₁·x) ⊗ W₃·x). The gate acts as a learned router. SwiGLU requires 3 matrices instead of 2, so to keep parameter count equal the intermediate dim is reduced to 2/3 of the standard 4× value. LLaMA uses d_ff = 11,008 (≈ 2/3 × 4 × 4,096).

def optimal_ND(C, lr=1e-2, steps=1000):
    """Find compute-optimal N, D for budget C FLOPs."""
    # FLOPs ≈ 6ND for training
    # Minimize L(N, C/(6N)) over N

    D = C / (6 * N)
    return N, D

BERT vs GPT: Encoder vs Decoder

Both BERT and GPT are transformer-based, but they differ in attention pattern and training objective. GPT (decoder-only) uses causal (unidirectional) attention and is trained with causal language modelling (CLM): predict the next token. This makes it naturally generative. BERT (encoder-only) uses bidirectional attention and is trained with masked language modelling (MLM): 15% of tokens are masked, and the model predicts them from both left and right context.

The bidirectional context gives BERT richer representations for classification tasks. But the autoregressive structure of GPT allows it to generate — and at scale, generation proves to be the more powerful paradigm. GPT-3 demonstrated that a large enough generative model could perform classification too, simply by framing the task as a generation problem (few-shot prompting). By 2023, decoder-only models dominated essentially every NLP benchmark.

T5 (Raffel et al., 2020) tried a third approach: encoder-decoder with a unified text-to-text framework. Every task is "translate" from input to output. T5 XXL (11B) was competitive with GPT-3 (175B) on many benchmarks, suggesting encoder-decoder efficiency advantages — but the paradigm hasn't scaled as cleanly as pure decoders.

KV Cache & Memory Efficiency

During inference, every input token's K and V projections are computed once and cached. When the next token is generated, only the new token's Q, K, V need to be computed — then the new K and V are appended to the cache, and attention is computed between the new Q and all cached K/V pairs. Without this cache, each decoding step would require O(N²) attention re-computation; with it, each step is O(N).

The memory cost is substantial. For LLaMA-3 8B with 32 layers, 8 KV heads, head dim 128, and 32-bit floats: one token occupies 32 × 8 × 128 × 2 × 4 = 262KB. At 128K context, that's 32GB just for the KV cache — equal to the model weights. This is why GQA (§1.6) is critical for long-context serving, and why vLLM's PagedAttention manages KV cache like OS virtual memory, enabling 24× higher throughput by eliminating fragmentation.

Flash Attention

Standard attention materialises the full N×N attention matrix in HBM (GPU global memory). For N=4096, that's 4096² × 4 bytes = 64MB per head per layer — but the bottleneck isn't compute, it's memory bandwidth. Reading and writing this matrix dominates runtime. Dao et al. (2022) observed that SRAM (the GPU's on-chip L1 cache) is 256KB but 19× faster than HBM. Flash Attention tiles Q, K, V into SRAM blocks, computes attention in tiles, and never materialises the full N×N matrix in HBM.

Flash Attention is exact, not approximate — it produces the same result as standard attention, just with radically fewer HBM accesses. The speedup comes purely from the memory hierarchy. FA-1 achieves 7.6× speedup and 9× fewer HBM reads/writes. FA-2 (2023) reorganises the parallelism across GPU threads, achieving another 2× speedup. FA-3 (2024) exploits H100 async tensor cores for yet more throughput.

Online softmax is the key trick. To tile attention without seeing the full row of scores at once, Flash Attention computes a running max and sum as it processes each tile, using the numerically stable identity: softmax(x) can be updated incrementally with correction factors. import numpy as np

def standard_attention(Q, K, V):
    """Standard: materialises full N×N matrix in memory."""
    d_k = Q.shape[-1]
    S = Q @ K.T / np.sqrt(d_k)          # N×N score matrix (HBM)
    P = np.exp(S - S.max(axis=1, keepdims=True))
    P /= P.sum(axis=1, keepdims=True)    # N×N softmax (HBM)
    return P @ V                         # N×d (HBM)

def flash_attention_sim(Q, K, V, block_size=2):
    """Flash attention: tiled, never stores full N×N."""
    N, d_k = Q.shape
    d_v = V.shape[1]
    O = np.zeros((N, d_v))              # output in HBM
    l = np.zeros(N)                     # running sum (l)
    m = np.full(N, -np.inf)             # running max  (m)
    hbm_ops = 0  # count HBM operations

    for j in range(0, N, block_size):   # KV block
        Kj = K[j:j+block_size]          # SRAM load
        Vj = V[j:j+block_size]          # SRAM load
        hbm_ops += 2 * block_size * d_k

        for i in range(0, N, block_size):  # Q block
            Qi = Q[i:i+block_size]      # SRAM load
            hbm_ops += block_size * d_k
            mi = m[i:i+block_size].copy()
            li = l[i:i+block_size].copy()
            Oi = O[i:i+block_size].copy()

            Sij = Qi @ Kj.T / np.sqrt(d_k)  # (Bi, Bj) in SRAM
            mij = Sij.max(axis=1)
            m_new = np.maximum(mi, mij)
            Pij = np.exp(Sij - m_new[:, None])
            lij = Pij.sum(axis=1)
            l_new = np.exp(mi - m_new) * li + lij
            O[i:i+block_size] = (np.exp(mi - m_new)[:, None] * Oi + Pij @ Vj)
            m[i:i+block_size] = m_new
            l[i:i+block_size] = l_new
            hbm_ops += block_size * d_v  # write back O

    O /= l[:, None]
    return O, hbm_ops

Alignment: RLHF, DPO & LoRA

A pretrained LLM is a raw document completer — brilliant but unpredictable. Alignment transforms it into a helpful assistant. The dominant approach, RLHF (Reinforcement Learning from Human Feedback, Ouyang et al. 2022), has three stages: (1) supervised fine-tuning on demonstration data, (2) training a reward model on human preference pairs, (3) optimising the LLM against the reward model using PPO while penalising KL divergence from the SFT baseline.

DPO (Direct Preference Optimisation, Rafailov et al. 2023) bypasses the reward model entirely. It derives a closed-form loss directly from preference pairs, using the observation that the optimal RLHF policy can be expressed analytically. DPO is simpler, more stable, and often matches RLHF quality — it has become the default for open-source alignment. Constitutional AI (Anthropic, 2022) adds a self-critique loop: the model generates responses, evaluates them against a written constitution, revises them, then produces preference pairs for the reward model — replacing human labellers with AI-generated preferences (RLAIF).

LoRA (Hu et al. 2021) is the dominant parameter-efficient fine-tuning method. Instead of updating the full weight matrix W ∈ ℝ^(d×d), LoRA freezes W and learns two low-rank matrices A ∈ ℝ^(d×r) and B ∈ ℝ^(r×d). The effective update is ΔW = BA with rank r ≪ d. For r=16, d=4096: LoRA uses 2×4096×16 = 131K parameters vs 16.7M for the full matrix — a 128× reduction. QLoRA (Dettmers 2023) combines LoRA adapters with NF4 quantized base weights, enabling fine-tuning LLaMA-3 70B on a single 48GB GPU.

# DPO Loss (Rafailov et al. 2023)
def dpo_loss(log_p_chosen, log_p_rejected, log_ref_chosen, log_ref_rejected, beta=0.1):
    """
    DPO: maximise margin between chosen and rejected log-ratios.
    beta controls how far from the reference policy we allow.
    """
    ratio_chosen   = log_p_chosen   - log_ref_chosen
    ratio_rejected = log_p_rejected - log_ref_rejected
    margin = beta * (ratio_chosen - ratio_rejected)
    loss = -np.log(1 / (1 + np.exp(-margin)))
    return loss, margin

Scale & Emergent Abilities

Scaling laws (Kaplan et al. 2020) showed that loss decreases smoothly as a power law in model size, dataset size, and compute — suggesting predictable improvement. But Wei et al. (2022) documented something more surprising: certain capabilities appear abruptly at a threshold scale, performing near-chance below that scale and near-perfectly above it. Multi-step arithmetic, chain-of-thought reasoning, and multi-language translation all show this step-function behaviour.

The interpretation is debated. Some argue emergence is an artefact of metric choice — a capability that improves smoothly in probability space looks discontinuous when measured by exact-match accuracy. Others argue it reflects genuine phase transitions in the model's internal representations. Either way, the engineering implication is clear: adding more parameters to an already-capable model can unlock qualitatively new behaviours that weren't predictable from smaller-scale experiments.

Mixture of Experts

A dense transformer activates all its parameters for every token. Mixture of Experts (MoE) replaces each FFN layer with N expert FFNs and a routing network. Each token is routed to only the top-K experts (typically K=2), so only a fraction of total parameters are active per forward pass. Mixtral 8×7B has 56B total parameters but only 13B active per token — matching LLaMA-3 13B in compute while having 4× more capacity for knowledge storage.

DeepSeek-V3 (2024) pushes this further: 671B total parameters, 37B active per token, with a novel auxiliary-loss-free load balancing strategy. The key engineering challenge is keeping experts evenly loaded — a "capacity factor" limits how many tokens each expert can handle per batch. Expert collapse (one expert dominating all tokens) is a common failure mode that's addressed with auxiliary load-balancing losses.

import numpy as np

def softmax(x):
    e = np.exp(x - x.max())
    return e / e.sum()

def moe_route(x, W_gate, top_k=2):
    """Route token x to top-k experts."""
    logits = x @ W_gate          # (n_experts,)
    probs = softmax(logits)
    top_idx = np.argsort(probs)[-top_k:][::-1]
    top_probs = probs[top_idx]
    top_probs /= top_probs.sum()  # renormalise
    return top_idx, top_probs, probs

Quantization & Limitations

Quantization reduces model precision to enable deployment on consumer hardware. INT8 (8-bit) has negligible quality loss and is supported by all major serving frameworks. INT4 (4-bit) saves 4× memory vs FP16 with 1–4% benchmark degradation depending on method. The key methods: GPTQ (second-order weight reconstruction per-layer), AWQ (protects the 0.1% of salient weights identified by activation magnitude), and GGUF (llama.cpp format, per-block mixed-precision, runs on CPU).

NF4 (NormalFloat4) is the theoretically optimal 4-bit format for normally distributed weights (which transformer weights approximately are). The 16 bin values are placed at the quantiles of a normal distribution rather than uniformly — more bins near zero, fewer at extremes. LLaMA-3 70B: 140GB in FP16, 35GB in 4-bit — fitting on a single A100 40GB.

def compute_nf4_bins(n=16):
    """NF4 bins: quantiles of N(0,1) for normally distributed weights."""
    quantiles = np.linspace(0.5/n, 1-0.5/n, n)
    from_scipy = [-1.0, -0.6962, -0.5252, -0.3949, -0.2844, -0.1848,
                  -0.0911, 0.0, 0.0796, 0.1609, 0.2461, 0.3379,
                   0.4407, 0.5626, 0.7230, 1.0]
    return np.array(from_scipy)

def llm_memory_budget(
    n_params_B,        # total parameters in billions
    n_layers,
    kv_heads,
    d_model,
    n_heads,

):
