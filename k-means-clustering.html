K-Means Clustering

Partitions data into K clusters by minimizing within-cluster variance — the most popular unsupervised algorithm.

20 examples across 13 datasets · Individual code snippets · LaTeX equations

Mathematical Foundation

\[\min \sum_{k=1}^{K}\sum_{x_i \in C_k}\|x_i - \mu_k\|^2\]

Minimize within-cluster sum of squares

\[\mu_k = \frac{1}{|C_k|}\sum_{x_i \in C_k}x_i\]

Centroid = mean of cluster members

\[\text{Silhouette}(i) = \frac{b(i)-a(i)}{\max(a(i),b(i))}\]

a=own cluster dist, b=nearest other cluster. [-1,1]

\[\text{Elbow: find } K \text{ where inertia curve bends}\]

Plot inertia vs K — bend = optimal K

Glossary of Technical Terms

Centroid
Center of a cluster — mean of all assigned points.
Inertia (WCSS)
Within-Cluster Sum of Squares. Lower=tighter clusters.
Silhouette
How similar to own cluster vs nearest other. +1=perfect, 0=boundary.
Elbow Method
Inertia vs K plot. 'Bend' suggests optimal K.
n_init
Random restarts (default 10). Avoids bad initializations.
K
Number of clusters — must specify in advance.
Lloyd's Algorithm
Assign→update centroids→repeat until stable.
Convergence
No points change cluster. Guaranteed but may find local minimum.

#1 ✓ Mall Customers

📦 Mall Customers (200 records)
💡 Customer segmentation
The classic clustering dataset. 2D (income × spending) makes clusters visually obvious. Real clustering problems have 10+ dimensions where visualization fails.
Features
CustomerIDGenderAgeAnnual Income (k$)Spending Score (1-100)
✅ Silhouette = 0.55 — Good Clusters
Silhouette score measures how similar data points are to their own cluster vs neighboring clusters
Silhouette = 0.20 means clusters are overlapping — points could reasonably belong to multiple clusters.
-1.0 to 1.0. Above 0.50 = reasonable structure. Below 0.25 = clusters are probably not meaningful.

import pandas as pd
df = pd.read_csv('Mall_Customers.csv')
X = df[['Annual Income (k$)', 'Spending Score (1-100)']].values

# --- DATA EXPLORATION ---
print(f'Dataset shape: {df.shape[0]} rows × {df.shape[1]} columns')
print(f'\nColumn names and types:')
print(df.dtypes.to_string())
print(f'\nFirst 10 rows — get a feel for the data:')
print(df.head(10).to_string())
print(f'\nBasic statistics:')
print(df.describe().round(2).to_string())
print(f'\nMissing values: {df.isnull().sum().sum()}')

# --- CLUSTERING ---
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import silhouette_score

# CRITICAL: Scale features before distance-based clustering
# Without scaling, features with large ranges dominate the distance calculation
X_cluster = df[['Annual Income (k$)', 'Spending Score (1-100)']].values
X_scaled = StandardScaler().fit_transform(X_cluster)
print(f'Clustering on {X_scaled.shape[1]} features, {X_scaled.shape[0]} samples')

model = KMeans(n_clusters=5, n_init=10, random_state=42)
labels = model.fit_predict(X_scaled)

# --- RESULTS ---
import pandas as pd
cluster_sizes = pd.Series(labels).value_counts().sort_index()
print(f'\nCluster sizes:')
for cluster_id, count in cluster_sizes.items():
    print(f'  Cluster {cluster_id}: {count} samples ({count/len(labels)*100:.1f}%)')

sil = silhouette_score(X_scaled, labels)
print(f'\nSilhouette Score: {sil:.4f}')
# → Measures how well each point fits its cluster vs nearest other cluster
# → Range: -1 (wrong cluster) to +1 (perfect). >0.5 = reasonable, >0.7 = strong

# --- VERIFY: What do the clusters actually represent? ---
if hasattr(X_cluster, 'shape') and X_cluster.shape[1] <= 10:
    cluster_df = pd.DataFrame(X_cluster, columns=['Income', 'Spending'])
    cluster_df['Cluster'] = labels
    print(f'\nCluster profiles (mean values per cluster):')
    print(cluster_df.groupby('Cluster').mean().round(2).to_string())

# ✓ VERDICT: K=5 income/spending segments
